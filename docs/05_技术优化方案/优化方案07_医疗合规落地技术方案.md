## 一、医疗数据分类分级标准

### 1.1 四级分类体系

| 级别 | 分类 | 定义 | 示例数据 | 安全要求 |
|------|------|------|---------|---------|
| **S4 特级** | 核心医疗数据 | 直接关联患者生命安全 | 药物处方、配伍禁忌规则、剂量计算 | AES-256加密存储+传输、双人审核、不可删除、操作审计 |
| **S3 高级** | 敏感个人健康信息 | 可识别个人的健康数据 | 诊断记录、检验报告、体检数据、基因信息 | AES-256加密、访问审计、脱敏展示、跨境禁止 |
| **S2 中级** | 业务运营数据 | 机构运营但不含个人健康 | 预约记录、排班数据、咨询记录、服务方案 | TLS传输加密、RBAC访问控制、日志审计 |
| **S1 普通** | 公开/内部数据 | 无个人隐私的业务数据 | 健康科普、通知公告、系统配置、模板 | 基本访问控制、防篡改 |

### 1.3 技术实现：数据分级标注

```java
// 数据分级注解（Spring Boot）
@Target({ElementType.FIELD, ElementType.TYPE})
@Retention(RetentionPolicy.RUNTIME)
public @interface DataClassification {
    DataLevel level();          // S1-S4
    boolean encrypt() default false;
    boolean audit() default false;
    boolean mask() default false;
    String maskPattern() default "";
}

public enum DataLevel {
    S1_PUBLIC,      // 普通
    S2_INTERNAL,    // 中级
    S3_SENSITIVE,   // 高级（敏感个人健康信息）
    S4_CRITICAL     // 特级（核心医疗数据）
}

// 使用示例
public class PatientRecord {
    @DataClassification(level = S4_CRITICAL, encrypt = true, audit = true)
    private String prescription;       // 处方数据

    @DataClassification(level = S3_SENSITIVE, encrypt = true, mask = true,
                        maskPattern = "NAME")
    private String patientName;        // 患者姓名

    @DataClassification(level = S2_INTERNAL, audit = true)
    private String appointmentId;      // 预约记录

    @DataClassification(level = S1_PUBLIC)
    private String templateContent;    // 模板内容
}
```

### 1.4 自动脱敏规则

| 字段类型 | 脱敏规则 | 原始值 | 脱敏后 |
|---------|---------|--------|--------|
| 姓名 | 保留首字 | 张三丰 | 张** |
| 手机号 | 保留前3后4 | 13812345678 | 138****5678 |
| 身份证 | 保留前3后4 | 330102199001011234 | 330***********1234 |
| 诊断 | 仅授权可见 | 2型糖尿病 | [已隐藏] |
| 地址 | 保留市级 | 杭州市西湖区XX路 | 杭州市*** |

```java
// 脱敏拦截器（MyBatis插件）
@Intercepts(@Signature(type = ResultSetHandler.class, method = "handleResultSets"))
public class DataMaskingInterceptor implements Interceptor {
    @Override
    public Object intercept(Invocation invocation) throws Throwable {
        Object result = invocation.proceed();
        if (result instanceof List) {
            ((List<?>) result).forEach(this::maskFields);
        }
        return result;
    }

    private void maskFields(Object obj) {
        for (Field field : obj.getClass().getDeclaredFields()) {
            DataClassification dc = field.getAnnotation(DataClassification.class);
            if (dc != null && dc.mask() && !hasPermission(dc.level())) {
                field.setAccessible(true);
                field.set(obj, MaskUtils.mask(field.get(obj), dc.maskPattern()));
            }
        }
    }
}
```

## 三、AI可解释性方案

### 3.1 可解释性分级

| 级别 | 场景 | 要求 | 实现方式 |
|------|------|------|---------|
| **强可解释** | 药物推荐、配伍检查 | 必须展示推理依据和引用来源 | 规则引擎+RAG来源追溯 |
| **中可解释** | 诊断辅助、护理建议 | 展示参考依据，标注AI生成 | RAG检索源+置信度评分 |

## 四、模型偏差检测方案

### 4.1 偏差检测维度

| 维度 | 检测方法 | 告警阈值 | 检测频率 |
|------|---------|---------|---------|
| **准确性偏差** | 人工标注样本对比 | 准确率<90%（S4场景<95%） | 每周 |
| **一致性偏差** | 同一输入多次调用对比 | 一致率<85% | 每日 |
| **时效性偏差** | 知识库更新后结果对比 | 过期引用率>5% | 知识库更新后 |
| **人群偏差** | 按年龄/性别/疾病分组评估 | 组间准确率差>10% | 每月 |
| **幻觉检测** | RAG来源核验 | 无源回答率>3% | 每日采样 |

### 4.2 自动化评估流程

```
┌──────────────────────────────────────────────┐
│           模型偏差检测 Pipeline                │
│                                               │
│  ① 采样                                      │
│     每日从生产请求中随机采样 1%                 │
│     按产品/场景/数据级别分层采样               │
│                                               │
│  ② 自动评估                                   │
│     - 幻觉检测：答案 vs RAG来源交叉验证        │
│     - 一致性检测：同题多次提交对比              │
│     - 拒答检测：模型是否正确拒绝越界问题         │
│                                               │
│  ③ 人工抽检                                   │
│     - S4特级场景：100%人工复核                  │
│     - S3高级场景：10%人工抽检                   │
│     - S2/S1场景：1%人工抽检                    │
│                                               │
│  ④ 告警与修正                                  │
│     - 指标异常 → 企微/PagerDuty告警            │
│     - 持续偏差 → 知识库更新/Prompt调优          │
│     - 严重偏差 → 自动降级为规则引擎兜底          │
└──────────────────────────────────────────────┘
```

### 4.3 评估指标看板（Grafana）

| 指标 | 计算方式 | 目标值 | 告警值 |
|------|---------|--------|--------|
| AI推荐采纳率 | 采纳次数/推荐次数 | >=70% | <50% |
| RAG检索命中率 | 有效检索/总检索 | >=85% | <70% |
| 幻觉率 | 无源回答/总回答 | <3% | >5% |
| 拒答准确率 | 正确拒答/应拒答 | >=95% | <90% |
| 用户负反馈率 | 踩/总反馈 | <10% | >20% |
