# 补充方案03：P3护理工作台离线模式技术方案

> 解决问题：P3是唯一有明确离线需求的产品，当前缺少完整技术方案
> 优先级：P1（2-3天工作量，影响患者安全）
> 版本规划：v1.0基础离线 → v2.0完整离线

---

## 一、需求背景

### 1.1 为什么P3必须支持离线

| 因素 | 说明 |
|------|------|
| **患者安全** | 护理操作不能因网络中断而停止，输液/用药/监护是持续过程 |
| **SLA 99.9%** | P3是全产品线最高可用性要求 |
| **物理环境** | 护士站/病房/治疗室/配液室分布在不同楼层，WiFi可能不稳定 |
| **移动场景** | v2.0+支持iPad平板，移动巡房时可能进入信号盲区 |

### 1.2 离线场景分析

| 场景 | 频率 | 持续时间 | 影响 |
|------|------|---------|------|
| WiFi短暂中断 | 高 | 10秒-5分钟 | 页面显示异常 |
| 网络设备故障 | 低 | 30分钟-2小时 | 无法获取新数据 |
| 服务器维护 | 极低 | 1-4小时（凌晨） | API不可用 |
| AI服务异常 | 中 | 5分钟-1小时 | AI功能不可用 |

## 二、功能分级方案

### 2.1 三级功能降级

| 级别 | 状态 | 可用功能 | 降级功能 | 不可用 |
|------|------|---------|---------|--------|
| **L0 正常** | 全部在线 | 所有功能 | — | — |
| **L1 弱网** | 网络不稳 | 缓存数据读取、本地规则引擎、历史记录查看 | AI响应变慢（超时→缓存结果） | 实时IoT数据刷新降为30秒→60秒 |
| **L2 离线** | 完全断网 | 核心护理操作（见下表）、本地规则引擎、离线数据录入 | AI功能全部→规则引擎降级 | 实时数据同步、AI Copilot对话 |
| **L3 AI不可用** | 网络正常但AI服务异常 | 全部非AI功能正常 | AI建议→规则引擎结果 | AI对话、AI预测 |

### 2.2 核心离线功能清单（L2级别）

| 功能 | 离线方案 | 数据来源 |
|------|---------|---------|
| **患者信息查看** | 本地缓存（最近24小时） | IndexedDB |
| **医嘱执行记录** | 离线录入→联网同步 | IndexedDB |
| **治疗提醒查看** | 本地缓存的当日排程 | IndexedDB |
| **配伍禁忌检查** | 本地规则引擎（严重禁忌库） | 预加载JSON |
| **剂量计算** | 本地计算（公式+规则） | 内置计算器 |
| **三查八对核验** | 本地数据比对 | IndexedDB缓存 |
| **高危药品提醒** | 本地规则库 | 预加载JSON |
| **护理记录填写** | 离线暂存→联网提交 | IndexedDB |
| **生命体征手动录入** | 离线暂存→联网提交 | IndexedDB |

### 2.3 不支持离线的功能

| 功能 | 原因 |
|------|------|
| AI Copilot对话 | 依赖大模型实时调用 |
| AI不良反应预测 | 依赖LSTM/Transformer模型 |
| AI排班优化 | 依赖服务端CSP算法 |
| IoT实时监测 | 依赖设备实时数据流 |
| 跨科室数据查询 | 依赖HIS实时接口 |

### 3.2 核心技术选型

| 组件 | 技术选型 | 选型理由 |
|------|---------|---------|
| 离线存储 | **IndexedDB + Dexie.js 4.x** | 支持大量结构化数据、索引查询、事务支持 |
| 离线缓存 | **Service Worker + Workbox 7.x** | 资源缓存、API离线拦截、后台同步 |
| 规则引擎 | **json-rules-engine** | 轻量级、纯JavaScript、无服务器依赖 |
| 网络检测 | **自定义心跳 + navigator.onLine** | 双重检测，避免误判 |
| 数据同步 | **自定义队列 + Background Sync API** | 联网后自动重放离线操作 |

### 3.5 Service Worker缓存策略

```javascript
// Workbox 缓存策略配置
workbox.routing.registerRoute(
  // 静态资源：CacheFirst（优先缓存）
  ({request}) => request.destination === 'script' ||
                 request.destination === 'style' ||
                 request.destination === 'image',
  new workbox.strategies.CacheFirst({
    cacheName: 'p3-static-v1',
    plugins: [new workbox.expiration.ExpirationPlugin({ maxEntries: 200, maxAgeSeconds: 7 * 24 * 3600 })]
  })
);

workbox.routing.registerRoute(
  // 关键API：NetworkFirst（优先网络，失败用缓存）
  ({url}) => url.pathname.startsWith('/api/nursing/patients') ||
             url.pathname.startsWith('/api/nursing/orders') ||
             url.pathname.startsWith('/api/nursing/schedules'),
  new workbox.strategies.NetworkFirst({
    cacheName: 'p3-api-critical',
    networkTimeoutSeconds: 5,
    plugins: [new workbox.expiration.ExpirationPlugin({ maxEntries: 500, maxAgeSeconds: 24 * 3600 })]
  })
);

workbox.routing.registerRoute(
  // AI API：NetworkOnly（不缓存，离线时走规则引擎降级）
  ({url}) => url.pathname.startsWith('/api/ai/'),
  new workbox.strategies.NetworkOnly()
);
```

## 四、数据冲突解决

### 4.1 冲突类型与解决策略

| 冲突类型 | 场景 | 解决策略 |
|---------|------|---------|
| **新增冲突** | 离线录入的护理记录，联网后发现同事已录入 | **服务端仲裁**：两条记录都保留，标记来源，由护士长审核 |
| **修改冲突** | 离线修改了执行状态，联网后发现医嘱已变更 | **服务端优先**：医嘱以服务端为准，本地修改标记为"待确认" |
| **时间冲突** | 离线期间记录的时间戳与服务端时间差异 | **本地时间戳 + 同步时间戳**双时间记录 |
| **删除冲突** | 离线期间操作的患者已出院 | **软删除**：本地记录保留，标记"患者已离院" |

### 4.2 同步队列设计

```javascript
// 离线操作队列数据结构
{
  id: autoIncrement,
  type: 'CREATE_NURSING_RECORD' | 'UPDATE_ORDER_STATUS' | 'RECORD_VITAL_SIGNS',
  payload: { /* 操作数据 */ },
  createdAt: '2026-03-19T10:30:00Z',  // 操作时间
  syncStatus: 'PENDING' | 'SYNCING' | 'SUCCESS' | 'CONFLICT' | 'FAILED',
  retryCount: 0,  // 最多重试3次
  conflictData: null,  // 冲突时存储服务端数据
  resolvedBy: null,  // 人工解决时记录处理人
}
```

### 4.3 同步流程

```
离线操作 → 写入syncQueue(PENDING)
          ↓
  [检测到网络恢复]
          ↓
  遍历PENDING队列（按createdAt排序）
          ↓
  调用服务端API → 成功 → 标记SUCCESS → 清理本地缓存
                → 409冲突 → 标记CONFLICT → 弹窗提示护士处理
                → 其他错误 → retryCount++ → 超过3次标记FAILED → 告警
```

## 五、本地规则引擎

### 5.1 配伍禁忌本地规则

```javascript
// 严重配伍禁忌本地规则（独立于AI服务）
const CRITICAL_CONTRAINDICATIONS = [
  { drug_a: "头孢类", drug_b: "含酒精制剂", severity: "CRITICAL", action: "BLOCK", reason: "双硫仑样反应" },
  { drug_a: "氨基糖苷类", drug_b: "利尿剂", severity: "HIGH", action: "WARN", reason: "耳毒性叠加" },
  // ... 200+ 高危规则
];

// 检查函数（纯本地，不依赖网络）
function checkContraindications(drugList) {
  const violations = [];
  for (let i = 0; i < drugList.length; i++) {
    for (let j = i + 1; j < drugList.length; j++) {
      const match = CRITICAL_CONTRAINDICATIONS.find(rule =>
        matchDrug(drugList[i], rule.drug_a) && matchDrug(drugList[j], rule.drug_b) ||
        matchDrug(drugList[i], rule.drug_b) && matchDrug(drugList[j], rule.drug_a)
      );
      if (match && match.severity === 'CRITICAL') {
        violations.push({ ...match, action: 'BLOCK' }); // CRITICAL级必须阻断
      }
    }
  }
  return violations;
}
```

### 5.2 剂量计算本地引擎

```javascript
// 剂量计算规则（纯本地）
const DOSE_RULES = {
  // 按体重计算
  weightBased: (weight, dosePerKg, maxDose) => Math.min(weight * dosePerKg, maxDose),
  // 按体表面积计算
  bsaBased: (height, weight, dosePerM2) => {
    const bsa = Math.sqrt((height * weight) / 3600); // Mosteller公式
    return bsa * dosePerM2;
  },
  // 肾功能调整
  renalAdjust: (baseDose, creatinine, age, weight, gender) => {
    const crcl = gender === 'M'
      ? ((140 - age) * weight) / (72 * creatinine)
      : ((140 - age) * weight) / (72 * creatinine) * 0.85;
    if (crcl < 30) return baseDose * 0.5;  // 严重肾功能不全
    if (crcl < 60) return baseDose * 0.75; // 中度
    return baseDose;
  }
};
```

## 六、网络状态检测

```javascript
class NetworkStatusManager {
  constructor() {
    this.status = 'ONLINE'; // ONLINE | WEAK | OFFLINE
    this.lastRTT = 0;
    this.heartbeatInterval = null;
  }

  start() {
    // 浏览器原生事件
    window.addEventListener('online', () => this.onNetworkChange());
    window.addEventListener('offline', () => this.setStatus('OFFLINE'));

    // 心跳检测（每10秒）
    this.heartbeatInterval = setInterval(() => this.heartbeat(), 10000);
  }

  async heartbeat() {
    try {
      const start = Date.now();
      const response = await fetch('/api/health', {
        method: 'HEAD',
        cache: 'no-store',
        signal: AbortSignal.timeout(5000)
      });
      this.lastRTT = Date.now() - start;

      if (this.lastRTT > 3000) {
        this.setStatus('WEAK');
      } else if (response.ok) {
        this.setStatus('ONLINE');
      }
    } catch {
      this.setStatus('OFFLINE');
    }
  }

  setStatus(newStatus) {
    if (this.status !== newStatus) {
      const oldStatus = this.status;
      this.status = newStatus;
      // 发布状态变更事件
      window.dispatchEvent(new CustomEvent('networkStatusChange', {
        detail: { from: oldStatus, to: newStatus, rtt: this.lastRTT }
      }));

      // 离线→在线：触发同步队列
      if (oldStatus === 'OFFLINE' && newStatus === 'ONLINE') {
        DataSyncManager.flushQueue();
      }
    }
  }
}
```
