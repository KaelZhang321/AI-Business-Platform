## 一、产品概述

### 1.1 产品定位

P2医生AI诊疗工作台是丽滋卡尔公司面向全职/坐诊医生打造的一站式智能诊疗平台。该平台以五栏布局（E/A/B/C/D区）为核心界面架构，深度融合AI诊断辅助、智能方案推荐、语音病历生成、疗效预测等9大核心AI能力，对接HIS/LIS/PACS等医疗信息系统，为医生提供从接诊到随访的全流程智能化支持。

**核心价值主张：**
- 诊断效率提升：AI辅助诊断覆盖率达到85%以上，减少漏诊误诊风险
- 方案制定加速：基于循证医学知识库的AI方案推荐，缩短方案制定时间50%
- 病历书写解放：语音转文字+AI结构化病历，病历书写时间减少70%
- 疗效可预测：AI疗效预测模型，治疗方案优化有据可依
- 合规保障：内置审方系统、配伍禁忌检测，降低用药风险

**产品边界：**
- 本产品为辅助诊疗工具，所有AI建议仅供参考，最终诊疗决策权归医生
- 不替代医疗设备的检查检验功能
- 不直接面向患者，患者端交互由P1健康管家AI工作台承担

### 1.2 医生用户画像

| 画像维度 | 详细描述 |
|---------|---------|
| **目标用户** | 丽滋卡尔全职/坐诊医生，约12-15人 |

#### 典型用户角色

**角色A：全科门诊医生（张医生）**
- 40岁，全科主治医师，日均接诊35人
- 需要快速浏览患者体检报告、既往病史，高效完成初步诊断
- 希望AI能帮助生成标准化病历，减少下班后加班补病历的情况

**角色B：专科坐诊医生（李医生）**
- 45岁，内分泌科副主任医师，每周坐诊3天
- 关注慢病管理效果追踪，需要AI辅助分析长期治疗数据
- 需要MDT会诊功能与其他专科医生协同

**角色C：体检中心医生（王医生）**
- 35岁，全科医师，主要负责体检报告解读和健康咨询
- 日均解读体检报告50+份，需要AI总检报告模块提升效率
- 需要快速对比历次体检指标，发现异常趋势

### 1.3 诊疗场景分析

#### 场景一：常规门诊接诊

```
流程：患者到达 → 叫号接诊 → 查看患者信息 → 问诊采集 → 体格检查 → AI辅助诊断
     → 开具检查 → 查看报告 → 确认诊断 → 制定方案 → 开具医嘱 → 医嘱确认 → 结束就诊
```

- **AI介入点**：问诊采集（语音转文字）、辅助诊断（鉴别诊断推荐）、方案制定（循证方案推荐）、开具医嘱（审方/配伍检测）
- **关键指标**：单次就诊时间 ≤ 20分钟，AI辅助诊断响应 ≤ 3秒

#### 场景二：MDT多学科会诊

```
流程：发起会诊 → 邀请专科医生 → 共享患者资料 → 多方讨论 → AI会诊辅助（文献/案例/方案）
     → 形成会诊意见 → 制定联合方案 → 会诊记录归档
```

- **AI介入点**：自动整理患者资料摘要、推荐相关文献和类似案例、生成会诊记录
- **关键指标**：会诊准备时间减少60%，会诊记录自动生成准确率 ≥ 90%

#### 场景三：体检报告解读

```
流程：获取体检报告 → AI总检分析 → 指标异常标注 → 历史对比 → AI四象限分类
     → 生成解读报告 → 医生审核确认 → 发送给客户
```

- **AI介入点**：AI总检报告自动生成、指标异常检测、历史趋势分析、四象限健康评估
- **关键指标**：单份报告AI生成时间 ≤ 30秒，指标异常检出率 ≥ 95%

#### 场景四：慢病随访管理

```
流程：随访提醒 → 查看患者近期数据 → AI疗效评估 → 方案调整建议 → 远程/电话随访
     → 更新治疗方案 → 设定下次随访
```

- **AI介入点**：疗效趋势预测、方案调整建议、用药依从性分析
- **关键指标**：慢病管理覆盖率 ≥ 90%，随访完成率 ≥ 85%

### 3.2 功能详细说明

#### 功能1：AI诊断评估

**功能描述：**
基于患者症状、体征、检验检查结果，结合医疗知识图谱和WiNGPT/YiduCore医疗模型，为医生提供AI辅助诊断建议。包含主诊断推荐、鉴别诊断列表、诊断依据分析、诊断置信度评分。

**交互流程：**
1. 医生在B区选中患者，系统自动加载患者历史数据
2. C区展开"AI诊断参考"卡片，自动显示初步AI分析结果
3. 医生可在A区AI助手中补充问诊信息（文字/语音输入）
4. AI实时更新诊断建议，显示TOP5可能诊断及置信度
5. 医生选择/修改诊断，点击"确认诊断"
6. 诊断结果自动写入患者电子病历

**AI能力：**
- 症状-疾病关联分析（基于WiNGPT医疗模型）
- 多维度鉴别诊断推理
- 检验指标异常与诊断的关联性分析
- 诊断置信度评分（0-100分）
- 相关指南/文献推荐

**输入：**
- 主诉、现病史、体格检查结果（文字/语音）
- 检验检查报告（LIS/PACS数据）
- 既往病史、用药史（HIS数据）
- 过敏史、家族史

**输出：**
- 主诊断推荐（ICD-10编码 + 诊断名称 + 置信度）
- 鉴别诊断列表（TOP5，含排除/支持依据）
- 建议追加检查项目
- 相关临床指南摘要
- 诊断依据分析报告

**验收标准：**
- AI诊断建议响应时间 ≤ 3秒
- TOP5诊断建议包含正确诊断的概率 ≥ 85%
- ICD-10编码准确率 ≥ 95%
- 支持至少500种常见疾病的辅助诊断
- 所有诊断建议标注"仅供参考，以医生判断为准"

#### 功能2：开具医嘱

**功能描述：**
医生开具药物医嘱、检查医嘱、护理医嘱、处置医嘱的核心操作功能。内置AI审方系统，实时检测药物配伍禁忌、剂量异常、过敏风险、重复用药等安全问题。

**交互流程：**
1. 医生点击"开具医嘱"或在诊断确认后自动跳转
2. C区展开医嘱编辑卡片，分为：药物医嘱/检查医嘱/护理医嘱/处置医嘱四个Tab
3. 药物医嘱：搜索药品（支持拼音/商品名/通用名），选择剂量、频次、疗程
4. AI实时审方：每新增一条医嘱，自动触发配伍禁忌检测
5. 异常警示：红色（禁忌，阻断）、橙色（慎用，需确认）、蓝色（提示）
6. 医生确认后提交医嘱，同步至HIS系统

**AI能力：**
- 实时配伍禁忌检测（药-药、药-食物、药-疾病）
- 剂量合理性校验（年龄、体重、肾功能调整）
- 重复用药检测
- 过敏交叉反应预警
- 基于诊断的常用处方推荐
- 个性化用药建议（基于患者基因药理学数据，如有）

**输入：**
- 确认的诊断信息
- 医生选择的药品/检查/处置项目
- 患者信息（年龄、体重、过敏史、肾功能等）
- 当前用药列表

**输出：**
- 结构化医嘱单
- 审方结果报告（通过/警告/阻断）
- 配伍禁忌详情（含参考文献）
- 剂量调整建议
- 同步至HIS系统的医嘱数据

**验收标准：**
- 配伍禁忌检测覆盖率 ≥ 98%
- 审方响应时间 ≤ 1秒
- 严重配伍禁忌（禁忌级别）必须阻断，不允许漏检
- 剂量异常检出率 ≥ 95%
- 医嘱格式符合《处方管理办法》规范
- 与HIS系统医嘱数据同步延迟 ≤ 5秒

#### 功能3：治疗方案制定

**功能描述：**
基于确认诊断，AI推荐循证治疗方案，包括药物治疗、非药物治疗（营养、运动、心理等）、随访计划。支持方案对比、方案定制化调整。

**交互流程：**
1. 诊断确认后，C区"AI方案推荐"卡片自动加载推荐方案
2. 展示1-3套治疗方案，包含药物方案、非药物干预、随访计划
3. 每套方案标注循证等级（A/B/C/D）、推荐强度、参考指南
4. 医生可对方案进行调整（增删药物、修改剂量、调整疗程）
5. 选择/定制方案后点击"采纳方案"
6. 方案自动拆解为医嘱，进入开具医嘱流程

**AI能力：**
- 基于最新临床指南的方案推荐（YiduCore循证知识库）
- 个体化方案调整（考虑合并症、禁忌症、患者偏好）
- 方案成本效益分析
- 方案间对比（疗效、副作用、费用）
- 历史方案效果回溯分析

**输入：**
- 确认诊断（ICD-10）
- 患者基本信息（年龄、性别、体重等）
- 合并症、禁忌症
- 既往治疗方案及效果
- 患者偏好（如有）

**输出：**
- 1-3套推荐治疗方案
- 每套方案包含：药物清单、剂量疗程、非药物干预、随访计划
- 循证等级和参考指南
- 方案对比分析表
- 预期疗效和风险评估

**验收标准：**
- AI方案推荐响应时间 ≤ 5秒
- 推荐方案与最新指南一致率 ≥ 90%
- 方案包含循证级别标注比例 = 100%
- 支持至少200种常见病的方案推荐
- 个体化调整因素覆盖 ≥ 5个维度

#### 功能4：MDT多学科会诊

**功能描述：**
支持医生发起/参与多学科会诊，AI自动整理患者资料、推荐相关专科、提供会诊辅助信息（文献、案例、方案建议），自动生成会诊记录。

**交互流程：**
1. 医生点击"发起会诊"，选择会诊类型（紧急/常规/远程）
2. 系统根据患者诊断推荐相关专科和可用医生
3. AI自动生成患者摘要（包含关键病史、检查结果、当前方案）
4. 会诊过程中AI实时提供辅助信息（文献检索、类似案例、方案建议）
5. 支持语音记录会诊讨论内容
6. 会诊结束后AI自动生成结构化会诊记录
7. 会诊意见自动关联到患者治疗方案

**AI能力：**
- 患者资料智能摘要生成
- 相关专科和医生推荐
- 实时文献检索和案例匹配
- 会诊讨论内容语音转文字
- 结构化会诊记录自动生成
- 会诊意见与治疗方案关联

**输入：**
- 患者完整病历资料
- 会诊申请信息（目的、紧急程度）
- 参会医生信息
- 会诊讨论内容（语音/文字）

**输出：**
- 患者资料摘要
- 会诊记录（结构化格式）
- 会诊意见和建议方案
- 会诊参考文献列表
- 后续随访计划建议

**验收标准：**
- 患者摘要生成时间 ≤ 10秒
- 会诊记录自动生成准确率 ≥ 90%
- 支持至少5人同时在线会诊
- 语音转文字准确率 ≥ 92%
- 会诊记录格式符合医院会诊管理规范

#### 功能5：疗效评价

**功能描述：**
基于患者治疗过程中的各项指标数据，AI辅助评估治疗效果，生成疗效评价报告，提供方案调整建议。

**交互流程：**
1. 在患者随访或复诊时，医生点击"疗效评价"
2. 系统自动拉取治疗前后的检验指标对比数据
3. AI生成疗效评价报告（指标变化趋势、改善/恶化分析）
4. 展示AI四象限分析图（疗效维度 vs 安全性维度）
5. AI给出方案调整建议（维持/加强/减量/更换）
6. 医生审核确认疗效评价，更新治疗方案

**AI能力：**
- 多指标疗效综合评分

## 四、A区AI诊疗助手详细设计

### 4.1 整体设计

A区宽度320px，垂直排列，从上到下分为：
- 顶部：当前患者信息摘要条（姓名、年龄、主要诊断）40px
- 对话区：AI对话窗口，占主体空间
- 底部：输入区（文字输入框 + 语音输入按钮 + 快捷指令）80px

### 4.2 对话设计

#### 对话模式

| 模式 | 触发方式 | 说明 |
|-----|---------|------|
| 自由对话 | 默认模式 | 医生可自由提问，AI基于患者上下文回答 |
| 诊断辅助 | 选择"AI诊断"功能 | AI主动引导式问诊，逐步收集信息 |
| 方案咨询 | 选择"方案制定"功能 | 围绕治疗方案的专项对话 |
| 文献检索 | 输入"查文献"关键词 | AI检索相关临床指南和文献 |

#### 对话消息类型

```
1. 文本消息 - 普通文字对话
2. 语音消息 - 语音转文字（保留原始音频可回放）
3. 诊断卡片 - AI诊断建议（含诊断名称、置信度、依据）
4. 方案卡片 - AI方案推荐（含药物、剂量、循证级别）
5. 用药卡片 - AI用药建议（含药品、剂量、频次、注意事项）
6. 警示消息 - AI风险预警（红/橙/蓝三级）
7. 引用消息 - 引用检验报告/病历/指南的片段
8. 操作建议 - AI建议下一步操作（如"建议开具XX检查"）
```

#### 对话上下文管理

- 每次选中新患者时，对话上下文自动切换
- 保留每个患者的独立对话历史
- 支持对话历史搜索和导出
- 敏感信息（如患者姓名）在对话记录中脱敏存储

### 4.3 语音输入设计

#### 硬件方案

| 设备 | 规格 | 数量 | 说明 |
|-----|------|------|------|
| 钉钉键（语音按键） | USB接口，物理按键 | 约12块 | 每个诊室1块，一键触发语音输入 |

#### 语音交互流程

```
1. 按下钉钉键 → 系统进入语音录制状态（界面显示录音指示器）
2. 医生口述内容 → Web Speech API实时转文字 → 显示在A区输入框
3. 松开钉钉键 → 停止录音
4. AI自动处理转文字内容：
   a. 如在"病历生成"模式：内容结构化为病历段落
   b. 如在"对话"模式：作为对话消息发送给AI
   c. 如在"医嘱"模式：解析为医嘱指令
5. 显示处理结果，医生可编辑确认
```

#### 语音识别优化

- 医疗专业词典：内置10000+医学术语（中文+英文缩写）
- 科室专属词典：根据当前登录医生科室加载对应词典
- 纠错机制：常见同音词自动纠正（如"心律"vs"心率"）
- 噪声抑制：诊室环境噪声过滤
- 断句优化：自动添加标点，智能分段

### 5.3 AI风险预警区域

## 八、九大核心AI功能详细说明

### 8.1 AI诊断辅助

| 维度 | 说明 |
|------|------|
| **算法方案** | 基于WiNGPT医疗大模型 + 医疗知识图谱的多模态诊断推理引擎。采用"症状-检查-诊断"三角交叉验证策略，结合临床决策树和贝叶斯推理网络 |
| **数据需求** | 症状库（5000+）、疾病库（3000+ICD-10编码）、症状-疾病关联矩阵、临床指南知识库（500+指南）、药品知识库（5000+药品） |
| **准确率目标** | TOP5诊断建议包含正确诊断 ≥ 85%，TOP1准确率 ≥ 65%，严重疾病漏诊率 < 3% |
| **性能指标** | 首次诊断建议响应 ≤ 3秒，增量更新 ≤ 1秒，并发支持 ≥ 15用户 |
| **合规要求** | 所有建议标注"仅供参考"，诊断记录可追溯，不直接出具诊断结论 |

### 8.2 AI方案推荐

| 维度 | 说明 |
|------|------|
| **算法方案** | 基于YiduCore循证知识库的方案检索+排序引擎，结合患者个体化特征进行方案定制。采用多目标优化算法平衡疗效、安全性、费用、依从性 |
| **数据需求** | 临床指南数据库、循证医学文献库、药品说明书数据库、历史方案效果数据、患者个体数据 |
| **准确率目标** | 推荐方案与最新指南一致率 ≥ 90%，个体化调整合理率 ≥ 85%，医生采纳率 ≥ 60% |
| **性能指标** | 方案推荐生成 ≤ 5秒，方案对比分析 ≤ 3秒 |
| **合规要求** | 所有方案标注循证等级和参考指南，方案调整记录完整可追溯 |

### 8.3 AI病历生成（语音转文字）

| 维度 | 说明 |
|------|------|
| **算法方案** | Web Speech API + 医疗领域ASR模型 + 医疗NLP结构化引擎。三阶段处理：语音→文字→结构化→质控 |
| **数据需求** | 医疗语音训练数据（需积累）、医疗专业词典（10000+术语）、病历模板库、质控规则库 |
| **准确率目标** | 语音识别准确率 ≥ 92%、医疗术语识别率 ≥ 88%、病历结构化准确率 ≥ 90%、质控覆盖率 = 100% |
| **性能指标** | 语音转文字实时（延迟 ≤ 500ms），结构化处理 ≤ 15秒/份，钉钉键响应 ≤ 200ms |
| **合规要求** | 符合《电子病历基本规范》，语音原始数据保留不少于30天，病历签名和修改痕迹完整保留 |

## 十二、典型诊疗AI交互场景

### 场景一：初诊糖尿病患者全流程

**患者背景：** 张女士，45岁，因"口渴多饮、多尿3个月，体重下降5kg"就诊。

**Step 1：接诊准备**
1. 医生登录P2工作台，查看B区"今日见诊"列表
2. 系统显示：09:30 张XX [待诊] 初诊 - 内分泌科
3. 医生点击患者条目，D区加载患者基本信息（首次就诊，信息有限）

**Step 2：问诊采集（语音病历）**
1. 医生按下钉钉键，开始语音录入
2. 口述："患者张某某，女性，45岁，主诉口渴多饮多尿3个月，伴体重下降约5公斤。既往体健，否认高血压糖尿病史，家族史父亲有2型糖尿病..."
3. A区实时显示语音转文字结果
4. 松开钉钉键，AI自动将口述内容结构化为主诉、现病史段落
5. C区"病历编辑"卡片显示AI生成的结构化病历
6. 医生审核确认，补充体格检查结果

**Step 3：AI辅助诊断**
1. AI自动分析已有信息，C区"AI诊断参考"卡片更新
2. 显示初步诊断建议：疑似2型糖尿病（置信度72%，因尚无检验结果）
3. AI建议追加检查：空腹血糖、HbA1c、OGTT、胰岛功能等
4. 医生点击"一键开具检查"，自动生成检验医嘱

**Step 4：获取检验结果**
1. 检验完成后，LIS系统推送报告至工作台
2. D区检验报告模块更新，AI自动解读：
   - FPG 8.2 mmol/L ↑（参考 3.9-6.1）
   - HbA1c 7.8% ↑（参考 < 6.5%）
   - 其他指标分析...
3. C区"AI诊断参考"卡片自动更新：
   - 2型糖尿病置信度提升至92%
   - 鉴别诊断更新

**Step 5：确认诊断 + 方案制定**
1. 医生点击"采纳"2型糖尿病诊断
2. C区"AI方案推荐"卡片自动加载治疗方案
3. 展示2套方案（方案一：二甲双胍+SGLT2i，方案二：二甲双胍单药）
4. 医生选择方案一，点击"采纳方案"
5. 方案自动拆解为药物医嘱

**Step 6：开具医嘱 + AI审方**
1. C区"医嘱管理"卡片展示拆解后的医嘱：
   - 二甲双胍 0.5g tid
   - 达格列净 10mg qd
2. AI审方系统实时检查：✅ 无配伍禁忌、✅ 剂量合理、✅ 无过敏风险
3. 医生确认提交医嘱，同步至HIS系统

**Step 7：病历完成 + 随访计划**
1. AI自动补充诊断和治疗方案到电子病历
2. 医生审核完整病历，确认签名
3. AI建议随访计划：4周后复查FPG+HbA1c
4. 随访任务自动添加到B区"待办事项"

### 场景二：MDT多学科会诊

**患者背景：** 李先生，60岁，2型糖尿病合并糖尿病肾病、冠心病，多科用药复杂。

**Step 1：发起会诊**
1. 内分泌科医生在E区点击"MDT多学科会诊"
2. AI自动推荐邀请科室：肾内科、心内科、营养科
3. 医生确认邀请，选择"常规会诊"，预约时间
4. AI自动生成患者摘要，推送给参会医生

**Step 2：会诊准备（AI自动）**
1. AI整理患者资料：
   - 基本信息 + 诊断列表
   - 关键检验指标趋势（eGFR变化、心脏标志物等）
   - 当前用药清单 + 药物相互作用分析
   - 各科治疗方案汇总
2. AI检索相关文献：
   - 糖尿病肾病最新治疗指南
   - 心肾综合征管理策略
   - 多药联用安全性文献

**Step 3：会诊讨论**
1. C区展开MDT会诊卡片，显示参会医生列表
2. 各科医生依次发表意见（支持语音输入，AI实时转文字）
3. AI实时提供辅助：
   - 当心内科医生提到"考虑加用利尿剂"时，AI提醒"注意与当前SGLT2i的叠加利尿效应"
   - 当肾内科医生建议"调整二甲双胍剂量"时，AI推送肾功能减退患者的剂量调整指南

**Step 4：形成会诊意见**
1. 会诊结束，AI自动生成结构化会诊记录
2. 包含：各科意见、联合治疗方案、用药调整、随访计划
3. 参会医生审核确认并签名
4. 会诊意见自动关联到患者治疗方案

### 场景三：体检报告AI解读与健康管理

**患者背景：** 王女士，50岁，年度体检完成，需要报告解读和健康建议。

**Step 1：AI总检报告生成**
1. 体检数据从体检系统同步至工作台
2. AI总检模块（外购）自动分析体检数据
3. 30秒内生成AI总检报告初稿
4. B区"待办事项"出现：1份AI总检报告待审核

**Step 2：医生审核AI总检报告**
1. 医生点击待审核报告，C区展开"体检报告"卡片
2. AI总检结论：
   - 空腹血糖6.5 mmol/L（糖尿病前期）
   - 总胆固醇5.8 mmol/L（偏高）
   - 甲状腺结节（4A级，建议随访）
   - 其他指标基本正常
3. D区显示历次体检对比：
   - 血糖：去年5.8→今年6.5（↑12%）
   - 胆固醇：去年5.2→今年5.8（↑11.5%）

**Step 3：AI四象限健康评估**
1. C区"体检对比"卡片展示AI四象限图
2. 王女士位于"B象限"（低风险/需关注区）
3. 风险趋势：向"A象限"（需干预区）移动趋势
4. AI建议：积极进行生活方式干预，3-6个月复查

**Step 4：健康管理方案**
1. A区AI助手生成健康管理建议：
   - 饮食方案：低糖低脂饮食，热量控制
   - 运动处方：每周150分钟中等强度运动
   - 复查计划：3个月后复查FPG+OGTT+血脂
   - 甲状腺结节：6个月后超声复查
2. 医生审核确认总检报告，签发
3. 健康建议通过P1健康管家推送给客户
