## 四、关键路径分析

### 关键路径

```
PRD编写(S9) → 设计评审(S10) → 前端框架(S11) → E区开发(S12) → A区AI助手(S13)
→ B区提醒中心(S14) → D区360视图(S15) → HIS对接(S16) → AI配液(S17)
→ AI预警+查房(S18) → 工作量+输液(S19) → 操作指引+卡片(S20)
→ 排班+交接(S21) → 剩余功能(S22) → 性能安全(S23) → 合规发布(S24)
```

### 关键路径风险节点

| 节点 | Sprint | 风险等级 | 风险说明 | 应对措施 |
|------|--------|---------|---------|---------|
| HIS数据对接 | S16 | **高** | HIS厂商配合度和接口稳定性 | 预留1周buffer，准备Mock数据降级方案 |
| AI不良反应预测模型 | S18 | **高** | 训练数据不足，模型准确率不达标 | 公开数据预训练+规则引擎兜底 |
| 多产品资源冲突 | S13-S20 | **高** | P1/P2/P3/P4并行开发资源紧张 | 明确优先级，P3按P1计划执行 |
| 年底收官 | S23-S24 | **中** | 安全审查发现问题或测试阻塞 | 持续安全检查，提前准备回归测试用例 |
