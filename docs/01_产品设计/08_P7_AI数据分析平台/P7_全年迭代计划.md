## 版本一：v0.5 - MVP验证（4月）

### 版本目标
验证Text2SQL核心技术可行性，实现基础自然语言查询功能，对接HIS和CRM两大核心数据源，面向运营经理小范围试用。

### User Stories

**US-001：作为运营经理，我希望通过自然语言查询预约数据，以便快速获取经营信息**
- 验收标准：
  - 支持自然语言输入查询（如"今天的预约量是多少"）
  - AI理解意图并生成SQL查询
  - 结果以数字+表格形式展示
  - 支持预约相关10个核心指标查询
  - 查询响应时间≤15秒
- 优先级：P0
- 故事点：21

**US-002：作为运营经理，我希望查询结果自动可视化，以便直观理解数据**
- 验收标准：
  - AI根据数据特征自动选择图表类型
  - 支持折线图/柱状图/饼图/表格4种基础类型
  - 用户可手动切换图表类型
  - 图表支持基础交互（鼠标悬浮显示数值）
- 优先级：P0
- 故事点：8

**US-003：作为运营经理，我希望基于查询结果继续追问，以便深入分析**
- 验收标准：
  - 支持基于当前结果追问（如"按门店拆分"）
  - 保持会话上下文
  - 追问结果在同一页面展示
- 优先级：P1
- 故事点：8

**US-004：作为运营经理，我希望查看查询历史，以便快速复查之前的分析**
- 验收标准：
  - 最近30条查询历史记录
  - 支持一键复问
  - 支持收藏常用查询
- 优先级：P1
- 故事点：5

**US-005：作为系统管理员，我希望配置数据源连接，以便接入业务系统**
- 验收标准：
  - 支持MySQL数据源连接配置
  - 连接测试功能
  - HIS和CRM系统成功接入
  - 元数据自动采集
- 优先级：P0
- 故事点：8

### Definition of Done (DoD)

- [ ] Text2SQL引擎基础功能可用
- [ ] 对预约相关问题的理解准确率≥75%
- [ ] HIS和CRM数据源成功接入
- [ ] 4种基础图表类型正常渲染
- [ ] 运营经理（3人+）试用并反馈
- [ ] 查询响应时间≤15秒

### 风险识别

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|----------|
| Text2SQL准确率不达标 | 高 | 高 | 限定查询范围、增加Few-Shot示例 |
| 数据源连接权限获取困难 | 中 | 高 | 提前协调IT部门开放只读副本 |
| 查询性能不达标 | 中 | 中 | 引入查询优化和结果缓存 |

## 版本二：v1.0 - 基础完善（5-6月）

### 版本目标
建立完整的指标体系和元数据管理，上线指标预警基础功能和自动日报，提升Text2SQL准确率，使平台达到日常可用状态。

### User Stories

**US-006：作为运营经理，我希望Text2SQL更准确更智能，以便信赖查询结果**
- 验收标准：
  - 查询理解准确率从75%提升至≥85%
  - 支持意图澄清（模糊问题主动询问）
  - 支持指标别名识别（营收=收入=销售额）
  - 查询响应时间优化至≤10秒
  - 支持收入类/客户类指标（扩展到30个核心指标）
- 优先级：P0
- 故事点：13

**US-007：作为运营经理，我希望系统对关键指标进行预警，以便及时发现异常**
- 验收标准：
  - 支持配置阈值预警规则
  - 提供6个预设预警模板
  - 预警触发后推送企微通知
  - 预警面板展示当前活跃预警
  - 预警检测频率：每30分钟
- 优先级：P0
- 故事点：13

**US-008：作为运营经理，我希望每天自动收到经营日报，以便快速掌握当日情况**
- 验收标准：
  - 每日18:00自动生成经营日报
  - 日报包含营收/预约/客户/运营4大模块
  - 日报自动推送到企微群
  - 日报PDF可下载
  - AI自动标注异常数据和变化
- 优先级：P0
- 故事点：13

**US-009：作为管理员，我希望管理指标体系和元数据，以便维护数据字典**
- 验收标准：
  - 指标字典管理（增删改查+分类）
  - 表字段与业务语义映射配置
  - 指标别名管理
  - 元数据变更自动通知
- 优先级：P1
- 故事点：8

**US-010：作为运营经理，我希望AI解读查询结果，以便理解数据含义**
- 验收标准：
  - 查询结果附带AI自然语言解读
  - 自动标注异常数据点
  - 同比/环比自动计算并展示
- 优先级：P1
- 故事点：8

**US-011：作为管理员，我希望配置数据权限，以便控制不同角色的数据访问范围**
- 验收标准：
  - 按角色配置数据域权限
  - 门店级数据隔离
  - 敏感字段自动脱敏
  - 查询审计日志
- 优先级：P0
- 故事点：8

### Definition of Done (DoD)

- [ ] Text2SQL准确率≥85%
- [ ] 指标预警基础功能上线，6个预设模板可用
- [ ] 日报自动生成并推送
- [ ] 指标体系覆盖30个核心指标
- [ ] 数据权限按角色控制
- [ ] 运营经理团队全员使用

### 风险识别

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|----------|
| 指标口径不统一导致数据不准 | 高 | 高 | 与业务方逐一确认每个指标定义 |
| 预警规则误报率高 | 中 | 中 | 初期设置保守阈值，逐步调优 |
| 日报内容不满足业务需求 | 中 | 中 | 业务方参与日报模板设计 |

## 版本三：v1.5 - 功能扩展（7-8月）

### 版本目标
上线体检看板日报和自动周报/月报，开发AI流程图绘制功能，接入财务系统数据源，使平台具备完整的数据分析和可视化能力。

### User Stories

**US-012：作为运营经理，我希望使用体检看板查看实时运营数据，以便实时掌控体检运营**
- 验收标准：
  - 实时数据区展示今日预约/到检/在检/完成数
  - 趋势分析区展示近30天核心指标趋势
  - 各门店对比排名
  - 支持日期范围和门店筛选
  - 数据自动刷新（可配置频率）
  - 全屏展示模式
- 优先级：P0
- 故事点：13

**US-013：作为管理层，我希望每周自动收到经营周报，以便掌握周度趋势**
- 验收标准：
  - 每周五17:00自动生成周报
  - 周报包含周趋势/门店对比/环比分析/关键事件
  - AI周度洞察报告
  - 推送企微+邮件
- 优先级：P0
- 故事点：8

**US-014：作为管理层，我希望每月自动收到经营月报，以便月度经营复盘**
- 验收标准：
  - 每月1日自动生成上月月报
  - 月报包含全景概览/深度分析/客户生命周期/目标完成率
  - AI月度深度洞察+下月预测
  - 月报可导出为PPT格式
- 优先级：P0
- 故事点：13

**US-015：作为运营经理，我希望AI生成业务流程图，以便可视化展示业务流程**
- 验收标准：
  - 通过自然语言描述生成流程图
  - 支持业务流程图/数据流图/漏斗图3种类型
  - 支持编辑调整（节点/连线/样式）
  - 支持导出PNG/SVG/PDF
- 优先级：P1
- 故事点：13

**US-016：作为管理员，我希望接入财务系统数据，以便分析收入和成本**
- 验收标准：
  - Oracle数据源连接支持
  - 财务系统成功接入
  - 收入/成本/利润相关指标可查询
  - 财务数据权限独立控制
- 优先级：P1
- 故事点：8

**US-017：作为运营经理，我希望自定义报表内容，以便生成符合需求的报表**
- 验收标准：
  - 支持选择报表模块和指标
  - 支持配置推送时间和接收人
  - 支持手动触发生成
  - 最多保存5个自定义报表模板
- 优先级：P2
- 故事点：8

### Definition of Done (DoD)

- [ ] 体检看板实时数据展示正常
- [ ] 周报/月报自动生成并推送
- [ ] AI流程图绘制基础功能可用
- [ ] 财务系统数据成功接入
- [ ] 看板全屏模式适配大屏投影
- [ ] 门店店长开始使用

### 风险识别

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|----------|
| 财务系统Oracle接入技术难度 | 中 | 中 | 预留技术攻关时间 |
| 体检看板实时性要求高 | 中 | 高 | 采用WebSocket+增量更新 |
| 月报PPT导出格式要求复杂 | 中 | 低 | MVP版先支持PDF，PPT后续迭代 |

## 版本四：v2.0 - 高级分析（9-10月）

### 版本目标
提升Text2SQL能力至高级分析水平（复杂查询、多表关联），上线AI深度洞察和自定义看板功能，面向管理层提供战略分析支持。

### User Stories

**US-018：作为管理层，我希望进行复杂的数据分析查询，以便深度理解经营状况**
- 验收标准：
  - 支持多表关联查询（如"体检客户中VIP客户的复购率"）
  - 支持时间窗口计算（如"最近3个月新客户的平均消费"）
  - 支持条件筛选+分组+排序的组合查询
  - Text2SQL复杂查询准确率≥80%
  - 查询响应时间：复杂查询≤20秒
- 优先级：P0
- 故事点：13

**US-019：作为运营经理，我希望AI主动发现数据洞察，以便不遗漏重要趋势**
- 验收标准：
  - AI每日自动扫描数据，发现异常和趋势变化
  - 洞察以卡片形式展示在首页
  - 洞察内容含数据事实+可能原因+建议行动
  - 支持对洞察进行标记（已知/有用/忽略）
- 优先级：P0
- 故事点：13

**US-020：作为运营经理，我希望创建自定义看板，以便按需监控关注的指标**
- 验收标准：
  - 拖拽式看板编辑器
  - 支持添加图表组件/数字卡片/文本/分割线
  - 图表数据源可关联"智能问数"的查询
  - 看板支持分享给其他用户
  - 看板自动刷新
- 优先级：P0
- 故事点：13

**US-021：作为管理层，我希望使用预警的高级规则，以便更精准地监控指标**
- 验收标准：
  - 支持同比/环比/趋势/波动预警规则
  - 预警规则可组合（如"取消率>15% 且 环比上升>30%"）
  - 预警分级（紧急/重要/一般）
  - 预警升级机制（未处理自动升级）
- 优先级：P1
- 故事点：8

**US-022：作为运营经理，我希望将查询结果分享给同事，以便协同分析**
- 验收标准：
  - 查询结果一键分享（生成分享链接）
  - 分享到企微群
  - 分享结果可固定为快照（不随数据变化）
  - 支持添加批注评论
- 优先级：P1
- 故事点：5

**US-023：作为市场经理，我希望分析营销活动效果，以便优化营销策略**
- 验收标准：
  - 营销系统数据接入
  - 支持查询获客成本/转化率/渠道ROI
  - 活动效果对比分析
  - 营销漏斗分析
- 优先级：P1
- 故事点：8

### Definition of Done (DoD)

- [ ] 复杂查询准确率≥80%
- [ ] AI洞察每日至少产出3条有价值发现
- [ ] 自定义看板编辑器可用
- [ ] 高级预警规则覆盖5种类型
- [ ] 管理层开始使用平台进行月度分析
- [ ] 市场经理可查询营销数据

### 风险识别

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|----------|
| 复杂查询Text2SQL准确率不足 | 高 | 高 | 增加训练数据，对复杂查询分步处理 |
| AI洞察质量参差不齐 | 中 | 中 | 设置洞察质量过滤阈值 |
| 自定义看板性能问题 | 中 | 中 | 限制单看板组件数量，优化渲染 |

## 版本五：v2.5 - 优化完善（11-12月）

### 版本目标
全面优化系统性能、AI模型准确率和用户体验，完善移动端体验，确保系统稳定性，交付年度最终版本。

### User Stories

**US-024：作为用户，我希望系统响应更快，以便高频使用无卡顿**
- 验收标准：
  - 简单查询响应≤10秒（从15秒优化）
  - 复杂查询响应≤20秒
  - 看板数据刷新≤5秒
  - 首屏加载≤2秒
  - 报表生成≤60秒
- 优先级：P0
- 故事点：13

**US-025：作为用户，我希望Text2SQL更准确，以便信赖查询结果**
- 验收标准：
  - 简单查询准确率≥92%
  - 复杂查询准确率≥85%
  - 支持更多自然语言表达方式
  - 错误查询时给出友好提示和建议
- 优先级：P0
- 故事点：13

**US-026：作为用户，我希望在手机上查看报表和看板，以便随时随地掌握数据**
- 验收标准：
  - 企微H5轻量版可查看日报/周报/月报
  - 可查看体检看板（简化版）
  - 支持简单的自然语言查询
  - 移动端加载≤3秒
- 优先级：P1
- 故事点：8

**US-027：作为管理员，我希望查看平台使用统计，以便评估产品价值**
- 验收标准：
  - 用户使用统计（日活/查询量/功能使用分布）
  - 查询准确率统计
  - 预警处理效率统计
  - 报表使用率统计
- 优先级：P1
- 故事点：5

**US-028：作为用户，我希望系统更稳定可靠，以便安心依赖使用**
- 验收标准：
  - 系统可用性≥99.5%
  - 错误率≤0.1%
  - 数据查询结果100%准确（与源系统一致）
  - 异常情况友好提示
- 优先级：P0
- 故事点：8

**US-029：作为运营经理，我希望流程图功能更完善，以便制作更多类型的图**
- 验收标准：
  - 新增支持组织结构图、决策树、甘特图

  - 导出质量优化

- 故事点：8

- [ ] 性能指标全面达标
- [ ] Text2SQL简单查询准确率≥92%
- [ ] 移动端轻量版上线
- [ ] 系统可用性≥99.5%
- [ ] 全年产品报告完成
- [ ] v2.5最终版发布
