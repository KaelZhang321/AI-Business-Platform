## 一、产品概述

### 1.1 产品定位

P7 AI数据分析平台是面向丽滋卡尔运营经理和管理层的智能数据分析工具，核心目标是实现"人人可问数据"。通过自然语言查询（Text2SQL）技术，让非技术人员无需编写SQL即可查询经营数据；通过指标预警、自动报表和智能看板等功能，将数据驱动决策的门槛降至最低，赋能全员数据化运营。

### 1.2 产品背景

丽滋卡尔在数据使用上面临以下挑战：
- **数据获取门槛高**：业务人员需通过IT部门或数据分析师获取数据，周期长、效率低
- **报表制作耗时**：日报/周报/月报需手工从多系统导出数据并整合，每周耗费大量人力
- **预警滞后**：关键指标异常发现依赖人工巡查，往往事后才知
- **数据孤岛**：各业务系统数据分散，缺乏统一的分析视角
- **可视化能力弱**：缺乏灵活的数据可视化和看板工具

### 1.3 目标用户

| 角色 | 人数(估) | 核心需求 | 使用频率 |
|------|----------|----------|----------|
| 运营经理 | 5-8人 | 日常经营数据查询、趋势分析 | 每日多次 |
| 门店店长 | 3-5人 | 门店运营指标、业绩对比 | 每日 |
| 管理层(VP/总经理) | 3-5人 | 经营概览、战略分析 | 每周 |
| 财务人员 | 2-3人 | 收入/成本/利润分析 | 每周 |
| 市场经理 | 2-3人 | 营销效果、客户分析 | 每周 |

### 1.4 核心价值主张

- **零门槛数据查询**：自然语言提问即可获得数据结果，无需SQL技能
- **自动化报表**：日报/周报/月报自动生成自动推送，解放人力
- **实时预警**：关键指标异常实时预警，变被动为主动
- **统一数据视图**：打通多系统数据，提供统一分析平台
- **AI驱动洞察**：AI自动发现数据异常和趋势变化，辅助决策

### 1.5 产品愿景

让丽滋卡尔的每一个经营决策都有数据支撑，打造"一问即答"的企业级智能数据分析平台。

## 三、5大功能详细设计

### 3.1 经营智能问数（Text2SQL）

**功能描述**：用户通过自然语言提问即可查询经营数据，系统自动将自然语言转换为SQL查询并返回可视化结果。

**支持的查询类型**：

| 查询类型 | 示例问题 | SQL模式 |
|----------|----------|---------|
| 指标查询 | "今天的预约量是多少" | SELECT COUNT |
| 趋势分析 | "最近一个月的营收趋势" | GROUP BY date |
| 对比分析 | "各门店本月业绩对比" | GROUP BY store |
| 排名查询 | "本月预约量TOP10的套餐" | ORDER BY LIMIT |
| 占比分析 | "各渠道预约来源占比" | GROUP BY + 百分比 |
| 同环比 | "本月营收同比增长多少" | 同期对比计算 |
| 筛选查询 | "VIP客户的平均消费金额" | WHERE + AVG |
| 明细查询 | "今天取消的预约列表" | SELECT + WHERE |

**核心指标体系**：

| 业务域 | 核心指标 | 计算口径 |
|--------|----------|----------|
| 预约管理 | 预约量、取消率、到检率、变更率 | 基于预约订单表 |
| 收入分析 | 营收总额、客单价、套餐收入、单项收入 | 基于财务收入表 |
| 客户分析 | 新客数、复购率、客户活跃度、客户生命周期价值 | 基于CRM客户表 |
| 运营效率 | 医生利用率、设备利用率、房间使用率 | 基于排班+预约表 |
| 市场营销 | 获客成本、转化率、渠道ROI | 基于营销+财务表 |

**交互设计**：
- **输入区**：对话式输入框，支持语音输入，自动补全常见问题
- **结果区**：数据+图表+AI解读三部分展示
- **历史区**：最近提问历史，支持一键复问
- **收藏区**：常用问题收藏，可设为定期自动查询
- **追问功能**：基于当前结果继续追问（如"按门店拆分看呢"）

**智能交互特性**：
- **意图澄清**：当问题模糊时，AI主动询问澄清（如"请问您是指预约量还是到检量？"）
- **推荐问题**：首次使用时推荐常见问题列表
- **联想查询**：输入过程中实时联想推荐
- **结果解读**：AI自动用自然语言解读数据结果
- **异常标注**：自动标注结果中的异常数据点

**性能要求**：
- NLU理解时间：≤2秒
- SQL生成时间：≤1秒
- 查询执行时间：≤5秒（简单查询）/≤15秒（复杂查询）
- 整体响应时间：≤10秒（简单）/≤20秒（复杂）

### 3.2 指标预警

**功能描述**：对关键经营指标设置预警规则，当指标异常时自动触发告警通知。

**预警规则配置**：

| 规则类型 | 说明 | 示例 |
|----------|------|------|
| 阈值预警 | 指标超过/低于设定阈值 | 日预约取消率>15% |
| 同比预警 | 指标同比变化超过阈值 | 月营收同比下降>10% |
| 环比预警 | 指标环比变化超过阈值 | 周预约量环比下降>20% |
| 趋势预警 | 连续N天指标持续下降/上升 | 连续5天客户活跃度下降 |
| 波动预警 | 指标波动超过正常范围 | 日预约量偏离均值2个标准差 |

**预设预警模板**：

| 预警名称 | 指标 | 触发条件 | 通知级别 |
|----------|------|----------|----------|
| 预约取消异常 | 日取消率 | >15% | 高 |
| 营收下滑预警 | 日营收 | 环比下降>20% | 高 |
| 客户流失预警 | 月活跃度 | 同比下降>15% | 中 |
| 到检率异常 | 日到检率 | <80% | 高 |
| 空位过多 | 时段空位率 | >70% | 中 |
| 医生超负荷 | 医生日预约 | >设定上限 | 高 |

**通知渠道**：
- 平台内通知（站内消息）
- 企业微信通知（文字/卡片）
- 短信通知（紧急预警）
- 邮件通知（日/周汇总）

**预警面板**：
- 当前活跃预警列表
- 预警历史记录
- 预警处理状态跟踪
- 预警规则管理入口

### 3.3 自动报表（日报/周报/月报）

**功能描述**：系统自动采集数据并生成标准化经营报表，按配置的时间和渠道自动推送给相关人员。

**日报内容**：

| 模块 | 指标 | 展示方式 |
|------|------|----------|
| 经营概览 | 营收/预约量/到检量/客单价 | 数字卡片+同比 |
| 预约分析 | 新增预约/取消/变更/来源分布 | 柱状图+饼图 |
| 客户分析 | 新客数/复购客户/VIP到访 | 数字+趋势 |
| 运营效率 | 医生利用率/设备使用率 | 进度条 |
| 异常事件 | 今日异常事件摘要 | 列表 |
| AI洞察 | AI自动发现的异常和趋势 | 文本 |

**周报内容（在日报基础上增加）**：
- 周趋势分析图表
- 各门店对比排名
- 环比变化分析
- 本周关键事件回顾
- AI周度洞察报告

**月报内容（在周报基础上增加）**：
- 月度经营全景概览
- 同比/环比深度分析
- 客户生命周期分析
- 目标完成率评估
- AI月度深度洞察
- 下月趋势预测

**报表配置**：
- 自定义报表内容模块（选择需要的模块和指标）
- 配置推送时间（日报默认18:00、周报默认周五17:00、月报默认次月1日）
- 配置推送渠道（企微群/邮件/平台通知）
- 配置接收人列表
- 支持手动触发生成

### 3.4 体检看板日报

**功能描述**：专门为体检业务打造的可视化看板，展示体检运营的核心指标。

**看板模块**：

**实时数据区**：
- 今日预约数 / 到检数 / 未到检数
- 当前在检人数 / 已完成人数
- 各检查室实时排队情况

**趋势分析区**：
- 每日预约量趋势（近30天）
- 到检率趋势
- 客单价趋势
- 套餐销售分布

**对比分析区**：
- 各门店预约量对比
- 各套餐销售量排名
- 各渠道来源对比
- 同比/环比关键指标

**体检流程看板**：
- 各检查项目的完成进度
- 检查项目平均耗时
- 等待时间最长的项目
- 异常项目标注

**看板交互**：
- 日期范围选择器
- 门店/科室筛选
- 数据下钻（点击指标查看明细）
- 全屏展示模式（大屏投影）
- 自动刷新（可配置刷新频率）

### 3.5 AI流程图绘制

**功能描述**：通过自然语言描述或数据分析结果，AI自动生成业务流程图、数据流图、组织结构图等可视化图表。

**支持的图表类型**：
- **业务流程图**：描述业务操作流程（如预约流程、客户服务流程）
- **数据流图**：展示数据在系统间的流转关系
- **组织结构图**：展示组织架构和汇报关系
- **决策树**：展示业务决策逻辑
- **甘特图**：项目进度可视化
- **漏斗图**：转化流程分析

**使用方式**：
1. **自然语言输入**：用户描述需要的流程图内容
   - 示例："画一个客户从预约到体检完成的完整流程图"
2. **数据驱动生成**：基于数据查询结果自动生成对应的可视化
   - 示例：查询各环节转化率后自动生成漏斗图
3. **模板选择**：从预设模板库选择并修改
4. **AI优化**：AI自动优化图表布局和样式

**编辑能力**：
- 节点增删改
- 连线调整
- 样式自定义（颜色/字体/形状）
- 导出为PNG/SVG/PDF

## 四、数据源接入方案

### 4.1 数据源清单

| 系统 | 数据库类型 | 核心数据表 | 接入优先级 |
|------|-----------|-----------|-----------|
| HIS系统 | MySQL 8.0 | 预约表、体检订单表、检查记录表 | P0 |
| CRM系统 | MySQL 8.0 | 客户表、沟通记录表、标签表 | P0 |
| 财务系统 | Oracle 19c | 收入表、成本表、发票表 | P1 |
| 排班系统 | MySQL 8.0 | 医生排班表、资源表 | P1 |
| 营销系统 | MySQL 8.0 | 活动表、投放表、转化表 | P2 |

### 4.2 接入方式

**方式一：直连查询**（只读副本）
- 连接各系统的只读从库
- 实时性好，延迟≤1分钟
- 需注意查询负载控制

**方式二：数据同步**（数据仓库）
- 通过ETL将数据同步到统一数据仓库
- 支持复杂跨系统联合查询
- 数据延迟T+1（日报场景可接受）

**方式三：API接入**
- 通过系统API获取数据
- 适用于无法直连数据库的第三方系统

### 4.3 元数据管理

**指标字典**：
- 统一管理所有业务指标的定义、计算口径、数据源映射
- 支持指标别名（如"收入"="营收"="销售额"）
- 指标分类管理（预约类/收入类/客户类/运营类）

**表字段映射**：
- 数据库表结构与业务语义的映射关系
- 支持可视化配置映射
- 新增数据表后自动提示配置映射

### 4.4 数据质量监控

- 数据完整性检查（空值率、异常值检测）
- 数据时效性监控（最新数据更新时间）
- 数据一致性检查（跨系统数据对比）
- 异常数据告警

## 五、安全与权限

### 5.1 数据权限模型

```
角色 ──→ 数据域权限 ──→ 可查询的数据范围
  │
  └──→ 功能权限 ──→ 可使用的功能模块
```

### 5.2 数据权限矩阵

| 角色 | 预约数据 | 收入数据 | 客户数据 | 人事数据 | 全门店 |
|------|----------|----------|----------|----------|--------|
| 运营经理 | 全部 | 汇总 | 全部 | 不可见 | 是 |
| 门店店长 | 本店 | 本店汇总 | 本店 | 不可见 | 否 |
| 管理层 | 全部 | 全部 | 全部 | 汇总 | 是 |
| 财务人员 | 汇总 | 全部 | 不可见 | 不可见 | 是 |
| 市场经理 | 汇总 | 汇总 | 全部 | 不可见 | 是 |

### 5.3 安全措施

- **查询审计**：所有查询操作记录审计日志（谁/何时/查了什么）
- **SQL注入防护**：AI生成的SQL经过安全审核引擎检查
- **数据脱敏**：敏感字段（手机号/身份证/地址）自动脱敏
- **查询限制**：单次查询结果上限10万条，防止全表导出
- **频率限制**：单用户每分钟最多10次查询
- **数据源只读**：所有连接均为只读权限，杜绝误操作

### 5.4 SQL安全审核规则

| 规则 | 说明 |
|------|------|
| 禁止DDL | 不允许CREATE/ALTER/DROP等操作 |
| 禁止DML | 不允许INSERT/UPDATE/DELETE操作 |
| 必须有WHERE | 大表查询必须有条件限制 |
| 限制JOIN数 | 最多允许5个表关联 |
| 超时限制 | 单次查询执行不超过30秒 |
| 结果集限制 | 最多返回10万行数据 |

## 六、典型应用场景

### 场景一：运营经理日常经营查询

**角色**：运营经理张总

**场景描述**：
周一上午，张总想快速了解上周的整体经营情况。

**操作流程**：
1. 打开AI数据分析平台，进入"智能问数"模块
2. 输入："上周的整体经营情况怎么样？"
3. AI自动理解意图，生成多维度查询：
   - 上周营收总额及同比变化
   - 上周预约量及到检率
   - 上周新增客户数
   - 各门店业绩排名
4. 结果以仪表盘形式展示，AI解读："上周营收环比增长8.3%，预约量同比增长12%，到检率93.5%保持稳定。XX门店表现突出，营收占比35%。"
5. 张总追问："哪些套餐卖得最好？"
6. AI立即展示上周套餐销售量TOP10排名和占比
7. 张总将结果一键分享到管理层企微群

**价值体现**：原来需要IT部门1天出报告，现在10分钟自主完成

### 场景二：管理层月度经营会议

**角色**：总经理李总

**场景描述**：
月底经营会议需要月度经营数据汇报。

**操作流程**：
1. 系统在月初自动生成月报，推送到管理层企微群和邮箱
2. 月报包含：
   - 月度经营全景概览（营收/利润/预约/客户）
   - 各门店业绩对比和排名
   - 同比/环比深度分析
   - 客户生命周期分析（新客增长/复购率/流失预警）
   - 目标完成率评估（含红黄绿灯标识）
   - AI深度洞察（自动发现的趋势和异常）
3. 会议中，李总直接用"智能问数"深入分析：
   - "为什么A门店本月营收下降了？"
   - "本月VIP客户续费率是多少？"
   - "营销活动ROI排名？"
4. AI即时给出分析结果和可能原因

**价值体现**：会议效率提升50%，数据驱动决策更精准

### 场景三：指标预警即时响应

**角色**：运营经理小王

**场景描述**：
某日下午，系统检测到预约取消率异常升高。

**操作流程**：
1. 14:00，预警系统检测到"日预约取消率"达到22%（阈值15%）
2. 系统立即推送预警通知到小王的企微：
   - 预警内容："今日预约取消率22%，超出阈值15%，较昨日上升65%"
   - AI分析："取消集中在14:00-16:00时段，主要涉及XX医生的预约，可能与XX医生临时请假有关"
3. 小王点击预警通知，跳转到平台查看详情
4. 通过"智能问数"追查原因："今天取消的预约都是哪些医生的？"
5. 确认原因后，协调处理受影响的预约
6. 预警事件标记为"已处理"并关闭

**价值体现**：异常发现从次日→实时，处理时效从小时级→分钟级
