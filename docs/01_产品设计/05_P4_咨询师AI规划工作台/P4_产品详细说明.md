## 一、产品概述

### 1.1 产品定位

咨询师AI规划工作台是丽滋卡尔AI产品矩阵中面向咨询师的智能化工作平台，定位于**客户消耗规划与升单转化的一站式智能辅助平台**。该产品通过AI技术赋能咨询师的日常工作，覆盖客户消耗规划（1+X）、预约对接、升单推荐、客户数据分析、话术生成等核心业务场景，旨在提升规划精准度、提升升单转化率、实现精准营销。

### 1.2 目标用户

| 属性 | 说明 |
|------|------|
| **用户群体** | 咨询师（约10-15人） |
| **岗位职责** | 客户消耗规划（1+X）、预约对接与管理、升单项目推荐、客户数据分析、话术生成与优化、业绩统计分析 |
| **使用场景** | 咨询室、办公区、外呼工位、客户接待区 |
| **技能水平** | 具备销售和客户服务经验，熟悉CRM系统操作，对数据分析工具有基础了解 |
| **使用频率** | 每日高频使用，贯穿客户接待、跟进、规划全流程 |

### 1.3 核心业务目标

| 目标 | 量化指标 | 说明 |
|------|---------|------|
| **提升规划精准度** | 消耗规划匹配率提升30% | AI基于客户画像生成1+X规划 |
| **提升升单转化率** | 升单成功率提升20% | AI精准推荐升单项目和话术 |
| **提升沟通效率** | 话术准备时间减少70% | AI自动生成个性化话术 |
| **精准客户管理** | 客户流失率降低15% | AI客户分层和流失预警 |
| **减少数据操作** | 数据整合时间减少80% | AI跨系统自动汇总 |
| **预约优化** | 预约冲突率降低50% | AI智能排期和冲突检测 |

### 1.4 技术架构

| 技术层 | 技术选型 | 用途 |
|--------|---------|------|
| **前端框架** | React 18 + TypeScript | 构建响应式五栏布局 |
| **UI组件库** | Ant Design 5.x | 标准化UI组件 |
| **拖拽布局** | React-Grid-Layout | 五栏拖拽自适应 |
| **数据可视化** | ECharts 5.x | 业绩图表、客户分析图、数据看板 |
| **后端框架** | Spring Boot 3.x | RESTful API服务 |
| **数据库** | MySQL 8.0 | 关系型数据存储 |
| **缓存** | Redis 7.x | 高频数据缓存、会话管理 |
| **消息队列** | RabbitMQ | 异步消息处理、预约推送 |
| **大语言模型** | DeepSeek / 通义千问 | AI对话、话术生成、客户分析 |

| **拖拽调节** | A/B区与C/D区的分隔线可拖拽调节 |

## 三、E区：岗位工具箱

E区位于页面最左侧，占据1/6宽度，是咨询师的核心操作入口。共包含13项功能。

### 3.1 消耗规划

| 属性 | 说明 |
|------|------|
| **功能描述** | 客户1+X消耗规划的智能管理。"1"代表基础健康管理套餐，"X"代表根据客户需求增加的个性化项目（如抗衰、皮肤管理、中医调理等）。系统根据客户健康数据、消费习惯、偏好等AI生成个性化消耗方案，支持方案对比和调整。 |
| **交互设计** | 点击后在A区打开消耗规划面板。顶部为客户选择器（搜索/下拉），中间展示AI生成的1+X规划方案：基础项目（1）和增值项目（X），每个项目显示名称、价格、预计消耗周期、优先级。底部为方案对比区域，支持多方案横向对比。右侧为AI推荐理由说明。 |
| **AI能力** | **AI方案生成** - 基于客户画像、健康状况、消费能力自动生成1+X消耗方案；**AI项目推荐** - 从产品库中筛选最匹配客户需求的项目；**AI消耗预测** - 预测客户未来3-6个月的消耗进度和到期时间；**AI方案对比** - 多方案的ROI对比分析。 |
| **验收标准** | 1.AI方案生成时间<5秒；2.方案推荐与客户需求匹配度≥80%；3.支持≥3个方案同时对比；4.消耗预测准确率≥75%；5.方案可导出为PDF格式。 |

### 3.2 预约管理

| 属性 | 说明 |
|------|------|
| **功能描述** | 客户预约的全流程管理，包括预约创建、查看、变更、确认、取消。支持见诊预约、治疗预约、体检预约等多种预约类型，提供日历视图和列表视图。 |
| **交互设计** | 点击后在C区显示预约管理面板。默认为日历视图（周视图），不同类型预约以不同颜色区分。支持拖拽创建新预约，点击已有预约查看详情。右上角快速创建按钮支持一键新建预约。冲突预约红色边框标注。 |
| **AI能力** | **AI冲突检测** - 自动检测时间冲突、医生排班冲突、设备冲突；**AI最优时间推荐** - 根据医生排班、客户偏好推荐最优预约时间；**AI批量确认** - 支持批量预约确认和自动推送提醒。 |
| **验收标准** | 1.预约冲突检测准确率100%；2.时间推荐响应<2秒；3.预约数据实时同步CRM；4.批量确认支持≥50条预约；5.预约变更通知推送成功率≥99%。 |

### 3.3 升单推荐

| 属性 | 说明 |
|------|------|
| **功能描述** | AI驱动的客户升单项目推荐。系统分析客户的消费历史、健康需求、消费能力、服务偏好等数据，精准推荐最适合的升单项目（套餐升级、新项目购买、VIP升级等）。 |
| **交互设计** | 点击后在C区显示升单推荐面板。顶部为客户筛选器（按升单潜力排序），中间为推荐项目列表，每个推荐包含：项目名称、推荐理由（AI生成）、预计转化率、客户匹配度评分。底部为历史升单成功案例参考。 |
| **AI能力** | **AI精准推荐** - 基于客户画像和消费模式预测最佳升单项目；**AI转化率预测** - 预测每个推荐项目的转化概率；**AI推荐话术** - 为每个升单推荐自动生成对应的沟通话术；**AI时机判断** - AI判断当前是否为最佳升单时机。 |
| **验收标准** | 1.升单推荐响应时间<3秒；2.推荐准确率（客户接受率）≥30%；3.每个推荐附带AI推荐理由和话术；4.支持推荐结果反馈闭环（接受/拒绝+原因）；5.转化率预测误差<15%。 |

### 3.4 客户分析

| 属性 | 说明 |
|------|------|
| **功能描述** | 客户多维度数据分析与洞察。支持客户分层分析（按价值/活跃度/消费能力）、购买潜力预测、流失风险评估、消费趋势分析等。提供可视化图表和AI分析报告。 |
| **交互设计** | 点击后在C区显示客户分析看板。顶部为分析维度切换（价值分层/活跃度/消费趋势/流失风险），中间为数据图表区域（散点图/柱状图/饼图），底部为AI分析结论和建议。支持下钻到单客户详情。 |
| **AI能力** | **AI客户分层** - 基于RFM模型自动将客户分为高价值/中价值/低价值/沉睡客户；**AI购买预测** - 预测客户未来3个月的购买概率和金额；**AI流失预警** - 识别有流失风险的客户并预警；**AI洞察报告** - 自动生成客户群体分析报告。 |
| **验收标准** | 1.客户分层计算时间<10秒（1000客户）；2.流失预测AUC≥0.75；3.购买预测准确率≥70%；4.分析图表支持≥5种可视化类型；5.AI报告生成时间<10秒。 |

### 3.5 话术生成

| 属性 | 说明 |
|------|------|
| **功能描述** | 根据客户画像、沟通场景、产品特点自动生成个性化沟通话术。支持初次接待话术、回访话术、升单话术、异议处理话术、促单话术等多种场景。 |
| **交互设计** | 点击后在A区显示话术生成面板。顶部为场景选择器（初次接待/回访/升单/异议处理/促单），中间为客户信息摘要（自动关联），底部为AI生成的话术内容（支持一键复制、编辑、收藏）。右侧为话术效果评分和历史话术参考。 |
| **AI能力** | **AI个性化话术** - 根据客户画像（年龄/职业/偏好/消费能力）定制话术内容和语气；**AI场景话术** - 针对不同沟通场景生成专业话术；**AI话术优化** - 基于历史成功话术的模式学习，持续优化生成质量；**AI多版本话术** - 一次生成多个版本供选择。 |
| **验收标准** | 1.话术生成时间<3秒；2.话术可读性和专业度评分≥8分/10分；3.一次生成≥3个版本供选择；4.支持话术收藏和个人话术库管理；5.话术效果反馈闭环（使用后标记效果）。 |

### 3.6 数据导出

| 属性 | 说明 |
|------|------|
| **功能描述** | 客户数据、业绩数据、分析报告的多格式导出功能。支持Excel、PDF、CSV格式导出，支持自定义导出字段和筛选条件。 |
| **交互设计** | 点击后弹出导出设置对话框。左侧为数据类型选择（客户数据/业绩数据/分析报告/预约数据），中间为字段选择和筛选条件设置，底部为格式选择和导出按钮。支持定时自动导出。 |
| **AI能力** | **AI自动汇总** - AI自动整合多数据源生成综合报告；**AI格式优化** - 根据数据特点自动优化导出格式和图表；**AI摘要生成** - 为导出报告自动生成数据摘要和关键发现。 |
| **验收标准** | 1.导出数据量≥10000条时响应<30秒；2.Excel/PDF/CSV三种格式正确无乱码；3.自定义字段支持≥50个可选字段；4.定时导出支持日/周/月频率；5.导出记录可审计。 |

### 3.7 预约确认

| 属性 | 说明 |
|------|------|
| **功能描述** | 预约确认的批量处理和自动化推送。支持批量电话确认、短信确认、企微消息确认等多种方式，记录确认结果（确认/取消/改期/无应答）。 |
| **交互设计** | 点击后在C区显示待确认预约列表。列表按确认截止时间排序，每条记录包含客户姓名、预约类型、预约时间、确认状态。支持批量勾选+一键推送确认消息。确认结果在列表中实时更新。 |
| **AI能力** | **AI自动提醒** - 根据预约时间自动生成提醒计划（提前1天/提前3小时）；**AI最佳时间** - 分析客户历史响应时间，推荐最佳确认联系时间；**AI话术匹配** - 根据客户特点推荐确认话术。 |
| **验收标准** | 1.批量推送≥50条确认消息时响应<10秒；2.推送成功率≥99%（短信/企微）；3.确认结果实时更新；4.支持至少3种推送渠道；5.确认记录完整可查。 |

### 3.8 客户标签

| 属性 | 说明 |

## 八、核心AI功能详细说明

### 8.1 AI消耗规划

| 维度 | 详细说明 |
|------|---------|
| **功能定位** | 根据客户情况自动生成1+X个性化消耗规划，提升规划精准度 |
| **算法方案** | 混合推荐算法：协同过滤（基于相似客户的消费模式）+ 内容推荐（基于客户健康数据和需求匹配）+ 规则引擎（基于产品适用规则和禁忌）。LLM负责将推荐结果转化为自然语言的规划方案，包括消耗理由和时间安排。 |
| **数据需求** | 1.客户基本信息和健康档案；2.历史消费和消耗记录；3.产品/套餐完整信息（功效/适用人群/价格）；4.相似客户群体的消费模式；5.医生建议和诊疗记录 |
| **性能指标** | 方案生成时间<5秒；推荐匹配度≥80%；方案接受率≥60%；系统支持同时生成≥3个方案 |
| **验收标准** | 1.覆盖100%在售产品/套餐的推荐能力；2.1+X方案包含消耗时间表和费用预算；3.支持方案自定义调整和重新生成；4.方案可导出为PDF/Word格式；5.方案效果可追踪（实际消耗vs规划对比） |

### 8.2 AI预约优化

| 维度 | 详细说明 |
|------|---------|
| **功能定位** | 智能推荐最优预约时间和医生，减少预约冲突，提升客户满意度 |
| **算法方案** | 约束优化算法：将预约排期建模为约束满足问题。约束条件包括：医生排班时间、设备可用时间、客户偏好时间、已有预约避让、最小间隔要求。优化目标：最大化客户偏好匹配+最小化空闲浪费+最大化医生利用率。LLM生成预约建议的自然语言说明。 |
| **数据需求** | 1.医生排班数据；2.设备/房间资源数据；3.客户时间偏好（历史统计）；4.已有预约数据；5.项目所需时间和设备需求 |
| **性能指标** | 推荐响应时间<2秒；冲突检测准确率100%；推荐接受率≥70%；支持同时推荐≥5个可选时段 |
| **验收标准** | 1.冲突检测覆盖时间/医生/设备三维度；2.推荐结果包含≥3个可选时段；3.支持一键预约；4.预约变更时自动推荐替代时段；5.预约数据实时同步CRM |

### 8.3 AI升单建议

| 维度 | 详细说明 |
|------|---------|
| **功能定位** | 分析客户需求和消费潜力，精准推荐最优升单项目，提升转化率 |
| **算法方案** | 多因素预测模型：基于梯度提升树（XGBoost）构建升单转化预测模型。特征工程包括：客户RFM评分、历史消费轨迹、剩余项目金占比、到店频率变化、同类客户升单成功率等。LLM生成升单推荐理由和配套话术。推荐排序综合考虑转化概率和客户终身价值(CLV)。 |
| **数据需求** | 1.历史升单成功/失败数据；2.客户消费轨迹和RFM评分；3.产品关联性数据（升单路径分析）；4.客户健康需求数据；5.同类客户群体的升单模式 |
| **性能指标** | 推荐响应时间<3秒；转化率预测AUC≥0.75；推荐接受率≥30%；每位客户推荐≥3个项目 |
| **验收标准** | 1.推荐项目附带匹配度评分和推荐理由；2.自动生成配套升单话术；3.支持推荐反馈闭环（采纳/未采纳+原因）；4.模型每月自动更新训练；5.升单成功后自动关联推荐记录 |

### 8.4 AI话术生成

| 维度 | 详细说明 |
|------|---------|
| **功能定位** | 根据客户画像和沟通场景生成个性化话术，提升沟通效率和专业度 |
| **算法方案** | 基于RAG增强的LLM生成架构。知识检索层：从话术库中检索相关成功话术案例和产品知识。提示工程层：根据客户画像（年龄/职业/消费层级/沟通偏好）和场景（初访/回访/升单/异议/促单）构建精准Prompt。后处理层：话术风格调整（亲切型/专业型/数据型），敏感内容过滤，格式化输出。 |
| **数据需求** | 1.历史成功话术案例库（≥1000条）；2.客户画像数据；3.产品知识库；4.沟通记录和效果反馈；5.行业最佳实践话术 |
| **性能指标** | 话术生成时间<3秒；一次生成≥3个版本；话术可读性评分≥8/10；用户满意度≥80% |
| **验收标准** | 1.支持≥5种沟通场景的话术生成；2.话术内容不含虚假承诺和违规用语；3.支持话术收藏和个人话术库管理；4.话术使用效果可追踪；5.支持话术模板自定义 |

### 8.5 AI客户分析

| 维度 | 详细说明 |
|------|---------|
| **功能定位** | 客户多维度分层分析和购买潜力预测，实现精准营销 |
| **算法方案** | 多模型组合分析。客户分层：基于RFM模型（最近消费时间、消费频率、消费金额）进行客户分层；流失预测：基于逻辑回归/随机森林的客户流失预测模型；购买预测：基于时间序列和客户特征的未来消费预测；CLV预测：客户终身价值预测模型。LLM负责将分析结果转化为可理解的洞察报告。 |
| **数据需求** | 1.客户全量消费记录；2.客户行为数据（到店、预约、沟通）；3.客户基本信息和标签；4.历史流失客户数据；5.行业benchmark数据 |
| **性能指标** | 分层计算时间<10秒（1000客户）；流失预测AUC≥0.75；购买预测准确率≥70%；分析报告生成<10秒 |
| **验收标准** | 1.客户分层支持≥4个层级；2.流失风险提前30天预警；3.分析结果可视化展示（散点图/热力图/饼图）；4.支持下钻到单客户分析；5.分析报告可导出分享 |

### 8.6 AI数据整合

| 维度 | 详细说明 |
|------|---------|
| **功能定位** | 跨系统客户数据自动整合汇总，减少人工操作 |
| **算法方案** | ETL + 实体对齐 + LLM增强。ETL层：定时从CRM、HIS、企微、ERP等多系统抽取客户相关数据。实体对齐层：基于客户ID和模糊匹配算法将多系统数据关联到统一客户档案。LLM增强层：对非结构化数据（沟通记录、备注）进行结构化提取，生成统一的客户档案。数据冲突时按数据源优先级和时间戳自动处理。 |
| **数据需求** | 1.CRM客户数据和消费记录；2.HIS诊疗记录和体检报告；3.企微沟通记录；4.ERP产品和价格数据；5.预约和排班数据 |
| **性能指标** | 数据同步延迟<5分钟；实体对齐准确率≥95%；非结构化数据提取准确率≥85%；系统可用性≥99.9% |
| **验收标准** | 1.整合≥4个数据源的客户数据；2.数据冲突自动处理规则覆盖90%场景；3.数据更新频率支持实时/准实时/定时三种模式；4.数据整合日志完整可审计；5.支持手动触发全量/增量同步 |

## 十、典型使用场景

### 场景一：VIP客户消耗规划与升单

**角色**：咨询师小陈，负责15位VIP客户

**流程描述**：

1. **9:00 登录系统** - 小陈登录咨询师AI规划工作台，B区智能提醒中心展示今日工作：待跟进客户20人，其中5位VIP需重点关注。AI风险预警提示：VIP客户张三消耗停滞已30天，流失风险评分75分（高风险）。

2. **9:10 客户分析** - 小陈点击张三的预警卡片，D区自动切换至张三的360视图。AI画像显示：65岁退休干部，VIP会员3年，剩余项目金58,800元，近3个月未到店消费。A区AI规划助手自动输出分析："张三近期消耗停滞可能原因：1.上次治疗效果不满意（沟通记录显示）；2.近期无预约；3.同期其他VIP客户活跃度正常。建议：主动回访了解情况，提供专属服务。"

3. **9:15 话术准备** - 小陈在E区点击"话术生成"，选择"挽留回访"场景。AI生成3个版本的话术："版本A（关怀型）：张先生，好久没见您了，最近身体还好吗？我们新推出了一个中医调理项目，非常适合您的体质...""版本B（价值型）：张先生，您卡里还有58,800元项目金...""版本C（活动型）：张先生，本月我们有一个VIP专属健康沙龙..."。小陈选择版本A并微调后保存。

4. **9:30 消耗规划** - 在E区点击"消耗规划"，AI自动生成张三的1+X方案：基础项目（1）：年度健康管理套餐续费；增值项目（X）：中医体质调理6次+抗衰疗程3次。方案总金额预估68,000元，预计消耗周期6个月。小陈微调方案后保存为"方案A"。

5. **10:00 电话回访** - 使用话术联系张三。通话中AI实时记录沟通要点。通话结束后AI自动生成通话摘要并保存至沟通记录。

6. **10:30 升单跟进** - 根据通话反馈，小陈在E区点击"升单推荐"。AI显示：推荐项目"中医调理金卡"，匹配度92%，预计转化率45%，推荐理由："张三在通话中表达了对中医调理的兴趣，且其体质报告显示气虚体质适合中医调理。"

**场景价值**：AI流失预警提前30天识别风险客户；个性化话术准备时间从30分钟缩短至3分钟；1+X消耗规划方案精准匹配客户需求；升单推荐基于数据驱动而非主观判断。

### 场景二：新客户接待与方案制定

**角色**：咨询师小李

**流程描述**：

1. **14:00 新客到店** - 系统提示：新客户王女士已到店签到。B区自动弹出接待提醒。D区显示从CRM同步的基础信息：32岁，金融行业，首次到店，通过市场活动引流。

2. **14:05 快速了解** - 小李在A区输入："帮我准备王女士的接待要点"。AI输出："王女士，32岁，金融从业者，首次到访。来源：3月线上活动。关注点预判：抗衰管理、亚健康调理（基于同画像客户分析）。建议接待策略：先了解需求，不急于推荐产品，重点建立信任感。推荐话术已生成。"

3. **14:10 面谈接待** - 小李使用AI推荐的接待话术进行面谈。面谈过程中通过语音记录要点。AI实时识别客户需求关键词："皮肤松弛""经常加班""睡眠不好""预算3-5万"。

4. **14:40 方案制定** - 面谈结束后，小李在E区点击"消耗规划"。AI基于面谈中识别的需求自动生成方案："基础项目（1）：年度健康管理基础套餐（含体检+营养咨询），X项目：光电抗衰4次+亚健康调理6次+睡眠改善方案。方案总金额42,000元，在客户预算范围内。"

5. **14:50 预约安排** - 客户有初步意向。小李在E区点击"预约管理"，AI推荐最优首诊时间："下周二上午10:00，李医生（抗衰专家），设备空闲，与客户偏好时段匹配。"一键预约成功。

6. **15:00 后续跟进** - 系统自动创建跟进计划：3天后回访确认预约，1周后见诊后跟进效果。AI为后续回访预生成了跟进话术。

**场景价值**：AI辅助的新客接待策略基于数据分析而非经验直觉；实时语音记录解放双手；AI方案自动匹配客户预算和需求；预约优化确保最佳的医生和时间匹配。

### 场景三：月度业绩分析与客户批量管理

**角色**：咨询师小王，月底业绩冲刺

**流程描述**：

1. **9:00 业绩查看** - 小王在E区点击"业绩统计"。C区显示本月业绩看板：签单总额320,000元（目标500,000元，达成率64%），升单2笔（目标5笔），新客转化3人。AI分析："当前达成率低于时间进度（月底前剩余3天），需要聚焦高潜力客户和到期客户。"

2. **9:10 客户筛选** - 在E区点击"客户分析"。AI自动筛选出高潜力客户名单："根据分析，以下8位客户有较高的短期购买概率：李四（消耗即将到期，续费概率85%）、赵五（上次沟通表达升级意向）、周六（VIP客户项目金余额较高可加单）..."

3. **9:20 批量话术准备** - 在E区点击"话术生成"，选择"促单"场景。AI为8位高潜客户分别生成个性化促单话术，小王快速浏览并微调后保存。

4. **9:30 批量预约确认** - 在E区点击"预约确认"，将明日到店的12位客户批量推送确认消息。AI根据每位客户的沟通偏好选择推送渠道（企微/短信/电话）。

5. **10:00 逐一跟进** - 按AI排序的优先级逐一联系高潜力客户。每次通话后AI自动记录并更新跟进状态。

6. **16:00 日终汇总** - AI自动生成今日工作汇总："今日联系8位客户，成功预约3位到店，1位表达明确购买意向（赵五，预计金额80,000元）。若赵五成交，月度达成率将提升至80%。"

**场景价值**：AI业绩分析实时量化差距和行动建议；智能筛选替代人工逐一判断；批量话术生成大幅提升工作效率；AI优先级排序确保有限时间投入到最高价值客户。
