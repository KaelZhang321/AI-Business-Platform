## 1. 产品概述

### 1.1 产品定位

P1 健康管家AI工作台是面向丽滋卡尔健康管家的一站式智能工作平台，采用六层架构（L1数据采集层、L2数据处理层、L3业务逻辑层、L4 AI能力层、L5展示交互层、L6运营分析层）和五栏布局（E/A/B/C/D区），将管家日常服务中的客户档案管理、到院接待、随访执行、消耗规划等核心场景整合到一个统一工作界面中。通过AI能力深度赋能，帮助健康管家提升服务效率、降低操作复杂度、提高客户满意度。

### 1.2 目标用户画像

| 维度 | 描述 |
|------|------|
| **用户群体** | 丽滋卡尔健康管家，约30-50人 |
| **年龄范围** | 25-40岁，以女性为主 |
| **教育背景** | 大专及以上，医学/护理/健康管理相关专业 |
| **技术水平** | 熟练使用智能手机和基础办公软件，对AI工具有一定接受度 |
| **工作模式** | 每人负责50-200名客户，日均处理15-30个服务事项 |
| **核心痛点** | 多系统切换频繁（HIS/CRM/体检/企微至少4个系统）、手动整理报告耗时、随访计划执行靠记忆、客户消耗规划缺乏数据支撑 |
| **期望价值** | 减少50%以上系统切换次数、AI辅助决策提升服务专业度、自动化提醒避免遗漏 |

### 1.3 使用场景

**场景一：晨间工作启动**
管家每日上班打开工作台，B区自动展示今日待办提醒（到院客户、需随访客户、即将过期套餐），C区展示AI生成的今日工作重点和客户洞察。

**场景二：客户到院服务**
客户到院后，D区自动拉取360度视图（基本信息、健康档案、消费记录、干预历史），A区AI Copilot根据上下文提供接待建议，E区提供快捷操作入口。

**场景三：体检报告解读**
管家收到客户体检报告，通过E区上传报告，AI自动解读关键指标异常、生成通俗易懂的解读文案、推荐干预方案，管家审核后通过企微发送给客户。

**场景四：周期性随访执行**
系统根据AI随访编排引擎生成的计划，在B区提醒管家执行随访任务，A区提供随访话术建议，完成后自动更新客户档案。

## 3. E区岗位工具箱

E区包含14项岗位工具，分为三组展示：

### 3.1 客户档案管理

**功能描述**：创建、编辑、查询客户健康档案，包括基本信息、疾病史、过敏史、家族病史、生活习惯等。

**交互流程**：
1. 管家点击E区"客户档案"图标
2. 弹出侧滑面板（宽度640px）
3. 顶部搜索栏支持姓名/手机号/会员号模糊搜索
4. 搜索结果列表展示，点击进入详情编辑
5. 档案编辑采用分步表单（基本信息→健康信息→服务信息）
6. 保存后自动同步至CRM系统

**AI增强能力**：
- 基于客户历史数据，AI自动补全缺失的档案字段
- 智能识别客户健康风险标签（如：高血压高风险、糖尿病前期）
- 自动关联同一家庭成员的档案

**输入**：客户基本信息、体检数据、就诊记录
**输出**：结构化健康档案、AI风险标签、家庭关联图谱
**验收标准**：档案创建成功率100%，搜索响应<500ms，AI标签准确率>85%

### 3.2 到院接待

**功能描述**：客户到院时一键触发接待流程，自动拉取客户信息、预约记录、今日项目安排，生成接待checklist。

**交互流程**：
1. 管家点击"到院接待"图标
2. 展示今日预约到院客户列表（按预约时间排序）
3. 选择客户后展示接待面板：左侧客户概要，右侧接待checklist
4. checklist包含：身份确认→签到→引导至相应科室→项目执行跟踪→满意度反馈
5. 每完成一步打勾确认，系统自动记录时间戳
6. 完成后生成接待报告

**AI增强能力**：
- 基于客户偏好和历史数据，AI推荐个性化接待话术
- 自动识别VIP客户并提醒升级服务
- 预测客户等待时间并提前通知

**输入**：预约记录、客户档案、项目安排
**输出**：接待checklist、接待报告、客户满意度评分
**验收标准**：接待流程启动<3s，checklist完整率>95%，客户识别准确率100%

### 3.3 随访执行

**功能描述**：执行系统生成的随访任务，支持电话、企微、短信多渠道触达，记录随访结果。

**交互流程**：
1. 管家点击"随访执行"图标
2. 展示待执行随访任务列表（按优先级和截止时间排序）
3. 选择任务后展示随访面板：客户信息+随访模板+历史随访记录
4. 选择触达渠道（电话/企微/短信）
5. 电话随访：集成VoIP一键拨号，通话中AI实时提示话术要点
6. 企微随访：AI生成推荐文案，管家编辑后发送
7. 完成后填写随访结果（客户状态、下次随访建议）
8. 系统自动更新客户档案和随访计划

**AI增强能力**：
- AI根据客户情况自动生成随访话术脚本
- 通话实时语音转文字，关键信息自动提取
- 随访结果智能分析，自动调整后续随访频率
- 识别客户情绪异常并预警

**输入**：随访任务、客户档案、随访模板
**输出**：随访记录、话术建议、情绪分析报告、下次随访计划
**验收标准**：随访任务加载<2s，AI话术生成<3s，语音转文字准确率>92%

### 3.4 消耗规划

**功能描述**：管理客户已购套餐的消耗进度，AI智能规划消耗节奏，避免过期浪费。

**交互流程**：
1. 管家点击"消耗规划"图标
2. 展示客户套餐消耗看板（按消耗率/到期时间排序）
3. 选择客户后展示详细消耗面板
4. 左侧：套餐项目列表+已消耗/剩余进度条
5. 右侧：AI推荐消耗计划（按时间轴展示）
6. 管家可调整AI建议的消耗计划
7. 确认后同步至预约系统

**AI增强能力**：
- 基于套餐有效期和客户时间偏好，AI自动编排消耗计划
- 消耗率过低自动预警
- 结合客户健康状态推荐最优消耗顺序
- 预测套餐到期风险并提前提醒

**输入**：套餐信息、消耗记录、客户时间偏好、健康档案
**输出**：消耗进度看板、AI消耗计划、预警提醒
**验收标准**：消耗数据实时同步延迟<5min，AI计划生成<5s，套餐到期预警提前30天

### 3.5 报告上传与解读

**功能描述**：上传客户体检报告（PDF/图片），AI自动解析并生成通俗解读文案。

**交互流程**：
1. 管家点击"报告解读"图标
2. 选择目标客户
3. 拖拽或点击上传报告文件（支持PDF/JPG/PNG，最大20MB）
4. 系统OCR识别+结构化提取（进度条展示）
5. 展示解读结果：左侧指标列表（异常项红色标记），右侧AI解读文案
6. 管家可编辑AI生成的解读文案
7. 一键通过企微发送给客户
8. 报告归档至客户健康档案

**AI增强能力**：
- OCR+NLP自动识别体检报告中的指标数值
- 结合客户历史数据纵向对比分析趋势
- 生成客户可读的通俗解读（支持多种风格：专业版/通俗版/关怀版）
- 自动标注需要关注的异常指标并推荐干预方案

**输入**：体检报告文件（PDF/图片）、客户历史体检数据
**输出**：结构化指标数据、AI解读文案、趋势分析图表、干预建议
**验收标准**：OCR识别准确率>95%，解读生成<10s，指标提取完整率>90%

### 3.6 干预方案管理

**功能描述**：基于客户健康数据和AI分析，制定并管理个性化健康干预方案。

**交互流程**：
1. 管家点击"干预方案"图标
2. 选择目标客户
3. 展示当前生效的干预方案列表
4. 新建方案：AI基于客户数据自动生成推荐方案
5. 方案内容：干预目标→干预项目→执行周期→效果指标
6. 管家审核编辑后激活方案
7. 方案执行跟踪：进度条+阶段性评估

**AI增强能力**：
- 基于客户健康画像和循证医学知识库，AI推荐干预方案
- 方案效果预测模型（预测干预后指标改善概率）
- 动态调整方案（根据阶段性评估自动优化）
- 类似客群成功案例匹配推荐

**输入**：客户健康档案、体检指标、诊断记录、可用服务项目
**输出**：AI推荐干预方案、效果预测、执行计划
**验收标准**：方案生成<8s，推荐方案采纳率>60%，干预效果跟踪覆盖率100%

### 3.7 内容生成器

**功能描述**：AI辅助生成各类客户沟通内容，包括随访话术、健康科普、节日关怀、活动推广等。

**交互流程**：
1. 管家点击"内容生成"图标
2. 选择内容类型（随访话术/科普文章/关怀消息/活动推广）
3. 填写关键参数（目标客户/客群、主题、语气风格、长度）
4. AI生成内容草稿（支持同时生成3个版本供选择）
5. 管家在富文本编辑器中编辑优化
6. 预览效果（模拟企微/短信展示）
7. 发送或保存至素材库

**AI增强能力**：
- 基于客户画像个性化内容生成
- 多版本生成供选择（正式/亲切/简洁三种风格）
- 自动配图推荐
- 敏感词检测与合规审查
- A/B版本效果追踪

**输入**：内容类型、目标客户/客群、主题关键词、风格偏好
**输出**：AI生成的内容草稿（多版本）、配图建议
**验收标准**：内容生成<5s，敏感词检出率100%，内容采纳率>70%

### 3.8 预约管理

**功能描述**：为客户创建、修改、取消预约，查看预约日历，管理预约冲突。

**交互流程**：
1. 管家点击"预约管理"图标
2. 展示日历视图（日/周/月切换），已有预约色块展示
3. 点击空闲时段创建新预约
4. 填写预约信息：客户→项目→医生/技师→时段→备注
5. 系统自动检测冲突并提示
6. 确认后发送预约通知（企微/短信）
7. 预约变更时自动通知相关方

**AI增强能力**：
- 智能推荐最优预约时段（考虑客户偏好、医生排班、项目时长）
- 预约爽约预测并提前干预
- 自动批量预约编排（适用于消耗规划场景）

**输入**：客户信息、项目信息、医生排班、时段偏好
**输出**：预约记录、日历视图、冲突提醒、预约通知
**验收标准**：预约创建<2s，冲突检测100%准确，通知送达率>99%

### 3.9 客户标签管理

**功能描述**：管理客户标签体系，支持手动标签和AI自动标签。

**交互流程**：
1. 管家点击"标签管理"图标
2. 展示标签管理面板：左侧标签分类树，右侧标签列表
3. 标签分类：基础属性标签、健康状态标签、消费行为标签、AI洞察标签
4. 支持手动给客户打标/去标
5. 查看AI自动生成的标签及其置信度
6. 基于标签筛选客户群

**AI增强能力**：
- 基于客户行为数据自动生成行为标签
- 基于健康数据自动生成风险标签
- 客户聚类分群
- 标签权重动态调整

**输入**：客户全维度数据
**输出**：标签列表、标签置信度、客群分类
**验收标准**：AI标签更新频率每日1次，标签准确率>85%，客群覆盖率100%

### 3.10 数据看板

**功能描述**：展示管家个人工作数据概览，包括工作量统计、客户服务覆盖率、KPI达成情况。

**交互流程**：
1. 管家点击"数据看板"图标
2. 展示个人工作数据面板
3. 核心指标卡片：今日服务客户数、待办完成率、随访完成率、客户满意度
4. 趋势图表：近7天/30天工作量趋势
5. 排行榜：管家团队排名（可选展示）
6. 导出周报/月报

## 5. B区智能提醒

### 5.1 今日待办

**功能描述**：汇聚管家当日需处理的所有待办事项。

**规则引擎设计**：
```
待办来源规则：
R1: IF 客户预约今日到院 THEN 生成"到院接待"待办（优先级：重要）
R2: IF 今日有计划随访任务 THEN 生成"随访执行"待办（优先级：根据客户等级）
R3: IF 客户套餐7天内到期 THEN 生成"消耗提醒"待办（优先级：紧急）
R4: IF AI检测到客户异常指标 THEN 生成"健康预警"待办（优先级：紧急）
R5: IF 客户生日在今日±3天 THEN 生成"生日关怀"待办（优先级：普通）
R6: IF 审批流程待处理 THEN 生成"审批处理"待办（优先级：根据超时时间）
```

**AI增强逻辑**：
- AI预测今日工作量并在晨间推送工作建议
- 基于历史完成率动态调整任务排序
- 智能合并相似任务（如同一客户的多个提醒合并展示）
- 预估每项任务耗时，帮助管家规划工作节奏

**优先级算法**：
```
Priority_Score = W1 × Urgency + W2 × Importance + W3 × Client_Level + W4 × Deadline_Proximity
其中：
- Urgency: 紧急程度（0-10），基于任务类型和截止时间
- Importance: 重要程度（0-10），基于任务对客户的影响
- Client_Level: 客户等级权重（VIP=10, 金卡=7, 银卡=5, 普通=3）
- Deadline_Proximity: 截止时间接近度（0-10），越近越高
- W1=0.3, W2=0.3, W3=0.2, W4=0.2（可配置）
```

### 5.2 随访计划

**功能描述**：展示近期随访计划时间线。

**规则引擎设计**：
```
R1: IF 客户术后N天 AND 需要定期随访 THEN 自动生成随访任务（频率由AI编排）
R2: IF 客户体检后7天 AND 未进行解读随访 THEN 生成提醒
R3: IF 客户30天未到院 AND 有未消耗项目 THEN 生成主动随访提醒
R4: IF 随访任务超期未完成 THEN 升级提醒优先级（每超1天提升10%）
R5: IF AI检测客户流失风险>0.7 THEN 生成紧急挽留随访任务
```

**AI增强逻辑**：
- AI随访编排引擎：基于客户状态、服务类型、历史随访效果自动编排最优随访节奏
- 智能推荐随访内容和渠道
- 预测随访成功率并优先安排高成功率任务
- 学习管家随访习惯，自动调整时间安排

### 5.3 预警通知

**功能描述**：展示需要管家关注的异常预警。

**规则引擎设计**：
```
R1: IF 客户关键指标连续恶化 THEN 健康预警（红色）
R2: IF 客户消耗率<30% AND 套餐剩余<90天 THEN 消耗预警（橙色）
R3: IF 客户满意度评分<3分 THEN 服务预警（橙色）
R4: IF 客户60天未互动 THEN 流失预警（黄色）
R5: IF 客户投诉未处理>24h THEN 投诉预警（红色）
R6: IF 管家KPI指标低于团队均值20% THEN 绩效预警（黄色）
```

**AI增强逻辑**：
- AI异常检测模型自动发现数据异常
- 预警智能聚合（同一客户多条预警合并展示原因链）
- 预警处理建议自动生成
- 预警趋势分析（是否在恶化或改善）

### 5.4 系统消息

**功能描述**：展示系统级通知、公告、版本更新等信息。

**规则引擎设计**：
```
R1: IF 系统有新公告 THEN 推送系统消息
R2: IF 工作台有新功能上线 THEN 推送引导消息
R3: IF 管家有新的学习任务 THEN 推送培训通知
R4: IF 后台数据同步异常 THEN 推送系统告警
```

**AI增强逻辑**：
- 消息智能分类和优先级排序
- 已读/未读自动管理
- 相关消息自动关联

## 8. 核心AI功能详细设计

### 8.1 AI报告解读

**功能描述**：基于大语言模型和医学知识库，自动解读客户体检报告和检验报告，生成通俗易懂的解读文案。

**算法方案**：
1. OCR层：阿里云OCR识别报告图片/PDF中的文字和表格
2. NLP层：结构化提取指标名称、数值、参考范围、单位
3. 知识增强层：Milvus向量数据库存储医学知识库，RAG检索相关医学解释
4. 生成层：DeepSeek/通义千问大模型生成解读文案
5. 审核层：规则引擎检查生成内容的医学合规性

**Prompt模板**：
```
你是一名专业的健康管理师，请基于以下体检报告数据为客户{客户姓名}生成解读报告：
{结构化指标数据}
客户历史数据：{历史指标趋势}
要求：1.用通俗易懂的语言 2.重点标注异常指标 3.给出生活建议 4.语气温暖关怀
输出格式：{指定格式}
```

**数据需求**：
- 体检报告原始文件（PDF/图片）
- 客户历史体检数据（至少近3次）
- 医学知识库（指标解读、疾病关联、生活建议）
- 客户基本信息（年龄、性别等影响参考范围的因素）

**性能指标**：
- OCR识别时间<5s
- 结构化提取时间<3s
- AI解读生成时间<10s
- 端到端总耗时<20s

**验收标准**：
- OCR识别准确率>95%
- 指标提取完整率>90%
- 解读内容医学准确性>95%（由医学顾问审核）
- 管家采纳率>80%
- 支持至少50种常见体检指标

### 8.2 AI干预方案

**功能描述**：基于客户健康数据和循证医学知识，自动生成个性化健康干预方案。

**算法方案**：
1. 客户画像构建：汇聚健康数据、消费数据、行为数据构建多维画像
2. 风险评估模型：基于指标异常程度和趋势计算各项健康风险评分
3. 干预匹配引擎：将风险项与可用服务项目进行匹配
4. 方案生成模型：LLM结合知识库生成结构化干预方案
5. 效果预测模型：基于历史数据预测方案效果

**数据需求**：
- 客户健康档案全量数据
- 机构可用服务项目库
- 历史干预方案及效果数据
- 循证医学知识库

**性能指标**：
- 方案生成时间<8s
- 方案个性化匹配度评分>7/10

**验收标准**：
- 方案推荐采纳率>60%
- 方案覆盖的风险项完整率>90%
- 干预后指标改善率>30%（3个月评估周期）

### 8.3 AI随访编排

**功能描述**：智能编排客户随访计划，自动确定随访时间、频率、内容和渠道。

**算法方案**：
1. 客户分层模型：基于RFM模型和健康状态将客户分为高中低维护层
2. 随访节奏引擎：根据客户层级、服务类型、术后恢复周期确定随访频率
3. 时间优化模型：基于客户历史互动数据预测最佳触达时间
4. 内容推荐引擎：根据随访目的和客户画像推荐随访内容
5. 渠道选择模型：基于客户渠道偏好和内容类型选择最优渠道

**数据需求**：
- 客户服务记录
- 历史随访数据（时间、渠道、结果、客户反馈）
- 客户互动行为数据
- 服务项目信息

**性能指标**：
- 计划生成时间<3s
- 计划覆盖率100%（所有活跃客户）

**验收标准**：
- 随访及时率>90%
- 随访完成率提升>20%
- 客户响应率提升>15%

### 8.4 AI内容生成

**功能描述**：基于大语言模型为管家生成各类客户沟通内容。

**算法方案**：
1. 模板库：维护各场景内容模板（随访话术、科普文章、关怀消息等）
2. 个性化引擎：基于客户画像进行模板变量填充和风格调整
3. LLM生成：DeepSeek大模型生成自然流畅的内容文案
4. 合规审核：敏感词过滤+广告法合规检查
5. 多版本输出：同时生成多种风格版本

**数据需求**：
- 内容模板库
- 客户画像数据
- 敏感词词库
- 历史内容效果数据

**性能指标**：
- 内容生成时间<5s
- 单次生成3个版本

**验收标准**：
- 内容采纳率>70%
- 敏感词检出率100%
- 客户打开率提升>10%

### 8.5 AI客户洞察

**功能描述**：基于客户全维度数据进行深度分析，生成客户洞察报告和行动建议。

**算法方案**：
1. 数据融合层：汇聚CRM、HIS、体检、企微多源数据
2. 特征工程：提取200+维度特征向量
3. 聚类分群：K-Means + DBSCAN双模型客户聚类
4. 预测模型：XGBoost流失预测、消费预测、活跃度预测
5. 洞察生成：LLM将分析结果转化为自然语言洞察

**数据需求**：
- CRM客户主数据
- 消费交易数据
- 到院行为数据
- 企微互动数据
- 体检健康数据

**性能指标**：
- 洞察报告生成<5s
- 预测模型AUC>0.8

**验收标准**：
- 流失预测准确率>75%
- 洞察报告可操作性评分>7/10（管家评价）
- 客户覆盖率100%

### 8.6 AI消耗规划

**功能描述**：智能规划客户套餐消耗节奏，最大化套餐利用率。

**算法方案**：
1. 套餐分析：解析套餐结构（项目、数量、有效期）
2. 消耗建模：基于项目间隔要求和客户时间构建约束规划模型
3. 优化求解：使用线性规划/遗传算法求解最优消耗排期
4. 风险评估：计算套餐到期风险和消耗率预测
5. 动态调整：根据实际消耗进度实时调整计划

**数据需求**：
- 套餐详情和有效期
- 已消耗记录
- 客户时间偏好
- 项目间隔要求
- 医生/设备排班

**性能指标**：
- 计划生成<5s
- 支持单客户最多10个套餐同时规划

**验收标准**：
- 套餐到期损耗率降低>30%
- 消耗排期合理性评分>8/10
- 预约转化率>80%

### 8.7 AI数据汇总

**功能描述**：自动汇总管家工作数据，生成可视化报表和AI分析报告。

**算法方案**：

3. 统计分析：时序分析、同环比分析、排名分析
4. 异常检测：基于统计方法+ML模型的异常指标检测
5. 报告生成：LLM将数据分析结果转化为工作报告

**数据需求**：
- 工作任务数据
- 服务记录数据
- KPI指标定义
- 团队基准数据

**性能指标**：
- 数据更新延迟<5min

**验收标准**：

系统15分钟前提醒，小李点击D区查看张女士360视图：健康档案显示近期血脂偏高（AI风险标签：血脂异常关注），套餐消耗进度62%，上次到院是2周前。A区AI Copilot主动提示："张女士今日预约光子嫩肤，建议询问近期血脂控制情况，上次推荐的低脂饮食方案执行如何。"

**Step 3 - 到院接待（10:00）**
张女士到院，小李点击E区"到院接待"启动接待流程。系统自动签到并推送欢迎消息。接待checklist显示：身份确认→引导至3楼医美中心→等候安排→项目执行。

**Step 4 - 服务中跟进（10:15-11:30）**
C区"客户动线监控"卡片实时显示张女士在3楼治疗室。AI预测项目时长75分钟，预计11:30完成。

**Step 5 - 服务完成后续（11:30）**
项目完成后，小李通过A区AI Copilot输入"/方案 张女士术后护理"，AI自动生成术后24小时护理注意事项和下次到院建议。小李审核后通过企微发送给张女士。

**Step 6 - 消耗规划更新（11:40）**
C区"AI消耗规划"卡片自动更新消耗进度至68%，AI建议下次预约时间和项目，小李确认后创建预约。
