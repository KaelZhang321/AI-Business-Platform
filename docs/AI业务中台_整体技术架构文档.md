## 一、架构设计总览

### 1.1 架构设计原则

| 原则 | 说明 | 落地方式 |
|------|------|---------|
| **统一底座** | 所有AI能力由技术中台统一提供 | 模型调用、知识检索、工作流编排均通过中台API |
| **松耦合** | 各应用层产品互不依赖，仅依赖中台 | 通过API网关解耦，独立部署独立迭代 |
| **可复用** | 中台能力组件化，各工作台按需组合 | 向量RAG、GraphRAG融合检索（详见优化方案11）、LLM、Agent框架（LangChain+LangGraph，详见补充方案02）、工作流等能力标准化封装 |
| **渐进式交付** | 从核心到扩展，从MVP到全功能 | 每个产品模块化迭代 |
| **数据驱动** | 所有AI交互数据可追踪、可评测、可优化 | 统一监控、A/B测试、效果评估闭环 |

### 1.3 核心交互流程

#### 流程1：知识问答（RAG）

```
用户: "公司的年假政策是什么？"
    │
    ▼
AI网关 → 意图分类（知识问答）
    │
    ▼
RAG引擎 → 检索知识库（Milvus + Elasticsearch）
    │
    ▼
大模型生成 → 返回答案 + 引用来源
    │
    ▼
前端渲染 → assistant-ui 流式输出
```

#### 流程2：数据分析（Text2SQL）

```
用户: "帮我查一下本周新增的客户有哪些？"
    │
    ▼
AI网关 → 意图分类（数据查询）
    │
    ▼
Vanna.ai → Text2SQL 转换
    │
    ▼
执行SQL → 数据仓库/业务数据库
    │
    ▼
动态UI生成 → json-render 渲染表格
```

#### 流程3：系统操作（待办任务）

```
用户: "帮我看看我有哪些待办任务？"
    │
    ▼
AI网关 → 意图分类（任务查询）
    │
    ▼
任务编排 → 聚合ERP/CRM/OA待办
    │
    ▼
动态UI生成 → json-render 渲染任务卡片
```

### 3.2 RAG检索模块

> **v1.5起升级为GraphRAG融合检索**（详见优化方案11）：在现有向量+关键词双路检索基础上，引入Neo4j知识图谱第三路检索，通过RRF（Reciprocal Rank Fusion）算法融合排序，显著提升医疗场景的检索准确率。

```python
# RAG Pipeline设计（v1.5升级为GraphRAG三路融合）
class RAGPipeline:
    """
    混合检索策略：
    1. 向量检索（Milvus）- 语义相似度
    2. 关键词检索（Elasticsearch）- BM25
    3. 图谱检索（Neo4j）- 知识图谱关系推理（v1.5+）
    4. 重排序（Reranker）- BGE-Reranker
    """

    def retrieve(self, query: str, top_k: int = 5):
        # 1. 向量检索
        vector_results = self.milvus.search(query, top_k=20)

        # 2. 关键词检索
        keyword_results = self.es.search(query, top_k=20)

        # 3. 图谱检索（v1.5+）
        graph_results = self.neo4j.search(query, top_k=20)

        # 4. 三路融合（RRF算法）
        fused_results = self.rrf_fusion(vector_results, keyword_results, graph_results)

        # 5. 重排序
        reranked = self.reranker.rerank(query, fused_results, top_k=top_k)

        return reranked
```

### 3.3 Text2SQL模块（Vanna.ai）

```python
# Vanna.ai集成设计
from vanna.ai import VannaAI
from vanna.postgres import Postgres

class BIQueryService:
    """
    Vanna.ai Text2SQL流程：
    1. 训练：导入数据库Schema + 样例问答对
    2. 推理：自然语言 → SQL
    3. 执行：安全SQL执行
    4. 渲染：结果转JSON Spec
    """
    
    def __init__(self, db_config):
        self.vanna = VannaAI(
            model="qwen2.5",  # 本地Qwen2.5
            api_key=os.getenv("API_KEY")
        )
        self.vanna.connect_to_postgres(**db_config)
    
    def train(self, training_data: List[dict]):
        """训练：导入Schema和问答对"""
        for item in training_data:
            self.vanna.train(
                question=item["question"],
                sql=item["sql"]
            )
    
    def query(self, natural_language: str) -> dict:
        """查询：自然语言转SQL并执行"""
        sql = self.vanna.ask(natural_language)
        result = self.execute_sql(sql)
        return self.to_json_spec(result)
```

    生成符合json-render规范的JSON Spec。
    
    组件目录：
    - Card: 卡片容器
